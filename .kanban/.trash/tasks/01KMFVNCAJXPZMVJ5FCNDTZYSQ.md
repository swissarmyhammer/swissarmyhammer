title: Board percent_complete recomputes and emits on task move

## What

When a task moves between columns (via drag, API, or any other process), the board's computed `percent_complete` field does not emit an `entity-field-changed` event. The UI never learns the board changed.\n\n**Backend fix** (`swissarmyhammer-kanban/src/task/mv.rs`): After `MoveTask` persists the position change, re-read the board entity (which triggers the `board-percent-complete` aggregate compute) and emit `entity-field-changed` for the board entity with the updated `percent_complete` value. This ensures any process that moves a task — UI drag, MCP agent, CLI — triggers a board-level update.\n\n**Frontend fix** (`kanban-app/ui/src/components/board-progress.tsx` + `nav-bar.tsx`): Replace the custom `BoardProgress` radial chart that reads from `board.summary` with a `<Field>` rendering the board's `percent_complete` field from the entity store. The existing `entity-field-changed` handler in `App.tsx` already updates entities in the store, so the Field will re-render automatically.\n\n### Files to modify\n- `swissarmyhammer-kanban/src/task/mv.rs` — emit board `entity-field-changed` after move\n- `kanban-app/ui/src/components/nav-bar.tsx` — use `<Field>` for percent_complete instead of custom `BoardProgress`\n- `kanban-app/ui/src/components/board-progress.tsx` — may be deleted or simplified to a display component\n\n## Acceptance Criteria\n- [ ] Moving a task to done emits `entity-field-changed` for the board entity\n- [ ] Board progress updates without UI calling refresh\n- [ ] Progress works when tasks are moved by any process, not just UI drag\n- [ ] Progress is rendered as a Field, not a bespoke component\n\n## Tests\n- [ ] Rust integration test: move task to done column → read board → `percent_complete.percent` increases\n- [ ] Rust integration test: move task out of done → `percent_complete.percent` decreases\n- [ ] Run: `cd swissarmyhammer-kanban && cargo nextest run move_task_updates_board_progress`\n- [ ] Run: `cd kanban-app/ui && npm test` — no UI regressions"
