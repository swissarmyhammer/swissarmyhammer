## What

No code spawns LSP server processes. `doctor.rs` detects whether `rust-analyzer`, `gopls`, etc. are installed but nothing acts on that. The `LspJsonRpcClient` is a stub that always returns `json!({})`.

**Key files:**
- `swissarmyhammer-code-context/src/lsp_communication.rs` — stub client, `send_request` never reads responses
- `swissarmyhammer-tools/src/mcp/tools/code_context/doctor.rs` — detects LSP availability
- LSP YAML configs in `swissarmyhammer-lsp/` crate

**Approach:** 
1. Use doctor results to determine which LSP servers to launch
2. Spawn LSP server processes per-language
3. Implement JSON-RPC initialize/initialized handshake
4. Keep server handles alive for the workspace lifetime
5. Graceful shutdown on workspace drop

## Acceptance Criteria
- [ ] `rust-analyzer` spawned and initialized for Rust workspaces
- [ ] LSP server process tracked and shut down cleanly
- [ ] Lifecycle logged at INFO level (start, ready, shutdown)
- [ ] Graceful fallback when LSP server not installed
