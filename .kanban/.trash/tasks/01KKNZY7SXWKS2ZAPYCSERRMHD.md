## What\nAdd a generic `OperationRegistry<C, E>` to `swissarmyhammer-operations` that maps `(verb, noun)` → deserialize + execute. Then build a kanban-specific registry in `swissarmyhammer-kanban`. This eliminates the ~420-line dispatch match block in the MCP tool and prevents duplication in kanban-cli.\n\n### Part A: Generic registry in `swissarmyhammer-operations`\n\n**Create:** `swissarmyhammer-operations/src/registry.rs`\n- `OperationRegistry<C, E>` with a `HashMap<(&str, &str), BoxedHandler>`\n- `register<T: DeserializeOwned + Execute<C, E>>()` — standard registration: deserializes params JSON into `T`, calls `processor.process(&t, ctx)`\n- `register_with_hook<T, F>(hook: F)` — registration with a pre-processing closure that can modify params before deserialization (for the 3 custom cases)\n- `dispatch(ctx, processor, verb, noun, params) -> Result<Value, E>` — look up handler, run it\n- Also expose `operation_metadata()` returning `&dyn Operation` instances for CLI generation\n\n**Modify:** `swissarmyhammer-operations/src/lib.rs` — add `pub mod registry;`, re-export\n\n### Part B: Kanban registry in `swissarmyhammer-kanban`\n\n**Create:** `swissarmyhammer-kanban/src/dispatch.rs`\n- `pub fn kanban_registry() -> OperationRegistry<KanbanContext, KanbanError>`\n- Registers all ~30 operations using plain `register` \n- Registers `AddTask`, `AddTag`, `AddActor` with hooks for: auto-assign actor, name/id fallback, ensure flag\n- `pub async fn dispatch_operation(ctx, op) -> Result<Value, KanbanError>` convenience function\n\n**Modify:** `swissarmyhammer-kanban/src/lib.rs` — add `pub mod dispatch;`, re-export\n\n### Part C: Wire up MCP tool\n\n**Modify:** `swissarmyhammer-tools/src/mcp/tools/kanban/mod.rs`\n- Replace the 420-line `execute_operation` match with a call to `kanban_registry().dispatch()`\n- Keep the plan notification logic and actor injection as pre-processing\n\n## Acceptance Criteria\n- [ ] `OperationRegistry` is generic and lives in `swissarmyhammer-operations` (no kanban dependency)\n- [ ] All ~30 kanban operations registered, including 3 with hooks\n- [ ] MCP kanban tool uses the registry — no more manual dispatch match\n- [ ] All existing MCP kanban tests pass unchanged\n\n## Tests\n- [ ] Unit test in `swissarmyhammer-operations`: registry with test operations, dispatch round-trip\n- [ ] Unit
