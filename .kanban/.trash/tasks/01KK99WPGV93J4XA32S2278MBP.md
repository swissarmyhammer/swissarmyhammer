Start rust-analyzer LSP server based on doctor availability check.

**Requirements:**
- Check if rust-analyzer is available (doctor.rs already does this)
- Spawn rust-analyzer process with proper arguments
- Connect to LSP server via stdio
- Set reasonable timeout/retry behavior
- Log all startup events for debugging

**Quality Test Criteria:**
1. Build succeeds: `cargo build 2>&1 | grep -c "error"` = 0
2. Doctor detects rust-analyzer: Call doctor on Rust project, verify `lsp_servers[0].installed == true` for rust-analyzer
3. Process spawns: Add integration test that:
   - Calls `start_lsp_server("rust")` on a Rust project
   - Verifies process spawns (check /proc or equivalent on macOS)
   - Check logs contain "rust-analyzer started" or similar
   - Verify stdin/stdout connected (no broken pipe errors)
4. Timeout handling: Test that server initialization has a reasonable timeout (5-30 seconds) and doesn't hang forever
5. Graceful error handling: Test that invalid/missing project gracefully logs error instead of panicking
6. Real project test: Verify no crash when starting LSP on actual swissarmyhammer-tools project
