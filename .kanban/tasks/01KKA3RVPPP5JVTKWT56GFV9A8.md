title: UpdateEntityField routes computed fields through DeriveHandler
---
Make `entity/update_field.rs` smart about computed fields. When the target field has `kind: computed`, look up its derive rule in the `DeriveRegistry` and call `apply()` instead of doing a direct field write.\n\n## Current behavior\n- `update_field` validates field name against entity schema\n- Reads entity, sets field value, writes entity\n- Has hardcoded tag auto-creation for body field updates (lines 94-114)\n\n## New behavior\n- Look up `FieldDef` for the target field via `FieldsContext::get_field_by_name()`\n- If `FieldType::Computed { derive }`: look up handler in `DeriveRegistry`, call `handler.apply(entity, schema, value)`. If handler doesn't exist or `writable() == false`, return error.\n- If normal field: existing direct-write path (unchanged)\n- Remove the hardcoded tag auto-creation block (lines 94-114) — this is now handled by the `ParseBodyTags::apply()` handler\n\n## Key: DeriveRegistry access\nThe `DeriveRegistry` needs to be accessible from `UpdateEntityField::execute()`. Options:\n- Store in `KanbanContext` (parallel to `FieldsContext`)\n- Store in `EntityContext` (next to `FieldsContext`)\n\n`KanbanContext` is the right place since derive handlers are kanban-domain specific (tag_parser, auto_color), not generic entity operations.\n\n## Files\n- Edit: `swissarmyhammer-kanban/src/entity/update_field.rs` — add computed field routing\n- Edit: `swissarmyhammer-kanban/src/context.rs` — expose `DeriveRegistry` from `KanbanContext`\n\n## Checklist\n- [ ] Add `derive_registry` field to `KanbanContext`\n- [ ] In `UpdateEntityField::execute()`, check if field is computed\n- [ ] If computed: look up handler, call `apply()`, write entity\n- [ ] If normal: existing direct write (unchanged)\n- [ ] Remove hardcoded tag auto-creation from body field update path\n- [ ] Existing tests still pass (update_body_auto_creates_tag_entities now goes through DeriveHandler)\n- [ ] New test: `update_field` on computed \"tags\" field calls derive handler\n- [ ] New test: `update_field` on computed field with no handler returns error\n- [ ] `cargo test -p swissarmyhammer-kanban` passes
