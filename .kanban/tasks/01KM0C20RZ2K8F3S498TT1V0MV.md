## What
Generate a bitmap image of the EntityCard to pass to `drag::Image::Raw()` so the OS drag ghost looks like the actual card, not a generic icon.

**Files:**
- `kanban-app/ui/src/components/board-view.tsx` or new utility — capture card DOM to image
- `kanban-app/src/commands.rs` — accept image data in `start_os_drag`

**Approach options (pick one during implementation):**

**Option A: html2canvas / dom-to-image (frontend capture)**
- When drag starts, use a library like `html2canvas` or `dom-to-image-more` to snapshot the card DOM element into a PNG/canvas
- Convert to base64 or raw bytes
- Pass to `start_os_drag` as the preview image
- Pros: pixel-perfect match to the real card. Cons: adds a dependency, async capture may delay drag start.

**Option B: Canvas rendering (frontend)**
- Render a simplified card representation onto a `<canvas>` element
- Export as PNG bytes
- Faster than DOM capture but won't be pixel-perfect
- Pros: fast, no dependency. Cons: visual fidelity limited.

**Option C: Pre-rendered template (Rust side)**
- Embed a simple card template image and overlay text using an image crate
- Pros: no frontend dependency. Cons: can't match dynamic styling.

**Recommendation:** Option A with `dom-to-image-more` (lightweight, well-maintained). The drag start already has a 5px activation distance which provides a small window for async capture.

The image needs to be passed as `drag::Image::Raw { data: Vec<u8>, width: f64, height: f64 }` — raw RGBA pixel data.

## Acceptance Criteria
- [ ] OS drag ghost shows a recognizable representation of the card being dragged
- [ ] Image includes card title text at minimum, ideally tags/colors too
- [ ] No visible delay when initiating drag (capture < 50ms)
- [ ] Image scales appropriately (not too large or blurry)

## Tests
- [ ] Manual test: drag a card — OS ghost looks like the card
