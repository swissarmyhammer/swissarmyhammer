title: 'Parametric command identity: scope chain accumulates entity commands'

## Problem\n\nThe current CommandScope system shadows commands by flat string ID. `entity.inspect` in a TagPill's FocusScope blocks `entity.inspect` in the parent EntityCard's FocusScope. But these are *different commands* — they inspect different entities. The scope chain should accumulate them, not shadow them.\n\n## Design\n\nCommand identity should be parametric — the verb + moniker target defines uniqueness:\n- `entity.inspect` + `tag:xyz` ≠ `entity.inspect` + `task:abc`\n- Shadowing only occurs when the full identity matches (same verb, same target entity)\n- The palette and context menu collect ALL entity commands up the scope chain\n\nA user clicking a task card, then opening the palette, should see:\n- Inspect tag (if a tag was last focused inside the card)\n- Inspect task\n- Inspect column\n- Inspect board\n\n## Approach\n\nOption A: Make command IDs include the moniker — `entity.inspect:tag:xyz`. Shadowing by full ID means different targets never shadow. Simple, but IDs become dynamic.\n\nOption B: Add a `target` field to CommandDef (the moniker). Shadowing checks both `id` and `target`. Commands with the same `id` but different `target` accumulate. More structured.\n\nOption C: Introduce a `group` concept — commands in the same group accumulate instead of shadowing. `entity.inspect` commands are in the \"inspect\" group.\n\nPreference: Option B — it preserves the existing ID system for non-entity commands (app.quit, app.undo) while adding parametric identity for entity-scoped commands.\n\n## Subtasks\n\n- [ ] Add optional `target?: string` (moniker) field to CommandDef\n- [ ] Update `resolveCommand` and `useAvailableCommands` — shadow only when both `id` AND `target` match\n- [ ] Update context menu collection to include all accumulated commands\n- [ ] Update command palette to show accumulated commands with target context in display\n- [ ] Each FocusScope's `entity.inspect` command includes `target: moniker`\n- [ ] Add FocusScope to ColumnView with inspect command targeting the column moniker\n- [ ] Add FocusScope at board level with inspect command targeting the board moniker\n- [ ] Connect entity focus to command palette — palette reads from focused entity's scope chain\n- [ ] Connect entity focus to global keydown handler\n- [ ] Clicking a tag focuses the tag (already happens via FocusScope)\n- [ ] Test: click card → palette shows inspect for tag (if focused), card, column, board\n- [ ] Test: right-click → context menu shows all inspects along the React ancestor axis\n- [ ] Run tests to verify no regressions
