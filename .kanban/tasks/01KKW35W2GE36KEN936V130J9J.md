title: WINDOW_COUNTER resets to 1 on restart — label collision with restored windows
---
kanban-app/src/commands.rs:20\n\n`WINDOW_COUNTER` is an in-process `AtomicU32` starting at 1. The `tauri-plugin-window-state` plugin restores saved windows on restart. If a previous session created windows `board-1` and `board-2`, the plugin will attempt to restore them. After restart, calling `create_window` again generates `board-1` — a label that already exists (being restored) — causing a Tauri error or silent failure.\n\nSuggestion: Start the counter high enough that fresh labels won't collide (e.g. use a timestamp-derived offset or scan existing window labels on startup). Alternatively, check for label existence before using it and increment past any collision."
