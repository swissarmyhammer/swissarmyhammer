title: Fix grid cell click region smaller than visible cell

**Bug**: In the kanban-app grid view, the click region (`<FocusScope>`) inside each grid cell is smaller than the visible cell — there's padding outside the focus region but inside the visible cell border. Users clicking near the edge of a visible cell expect focus to land on that cell, but the click misses the focus region.

**Files to investigate:**
- `kanban-app/ui/src/components/data-table.tsx` — `GridCellFocusable` and `DataBodyCell`
- `kanban-app/ui/src/components/grid-view.tsx`

**Fix**: Make the focusable element fill its visible cell. Pick whichever option fits the existing styling pattern:
1. Move `<FocusScope>`'s rendered element OUT to the cell wrapper (so it IS the visible cell box).
2. Make the focused element fill its parent (`w-full h-full`) and pull padding inward.

**Acceptance criteria:**
- Visible cell boundary equals click target
- `getBoundingClientRect()` of focusable matches visible cell rect
- No layout regression in row-header cells, header cells, surrounding chrome
- `pnpm -C kanban-app/ui exec tsc --noEmit` clean
- Existing grid tests stay green (or assertions updated to match new dimensions)
