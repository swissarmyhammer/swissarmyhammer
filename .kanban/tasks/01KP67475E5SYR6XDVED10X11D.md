## What

`ReferenceSelectEditor` at `/Users/wballard/github/swissarmyhammer/swissarmyhammer-kanban/kanban-app/ui/src/components/fields/editors/reference-select-editor.tsx` is the editor wired behind `editor: select` for reference fields (`field.type.entity` set). `SelectEditorAdapter` at `/Users/wballard/github/swissarmyhammer/swissarmyhammer-kanban/kanban-app/ui/src/components/fields/registrations/select.tsx` routes `position_column` and `project` through it.

That editor is built on shadcn Combobox + native `<input>` with custom keyboard nav — **not CM6**. Meanwhile `MultiSelectEditor` (used by `tags`, `depends_on`, `assignees`) is CM6-backed and shares the mention-pill infrastructure with `TextEditor` / the description editor. The single-select reference path is the only mention-style editor left fragmented across two UI frameworks.

The design rule (from the `pill-via-cm6` project: see `.kanban/tasks/01KNZ42426TGY4AVSDMQMRFQGS.md`, `.kanban/tasks/01KNZ45W47G3CYH64ZVV5KXTSP.md`, `.kanban/tasks/01KNZ48K8GBVSS0X8887ZQTB61.md`) is "exactly one way pills render in the app" — every mention-based editor must use the same CM6 foundation, sharing `createMentionCompletionSource`, `createMentionAutocomplete`, `createMentionDecorations`, and `buildSubmitCancelExtensions`.

### Historical note

A prior attempt (`SelectReferenceEditor`) was implemented as a CM6 single-select and then deleted during `01KP3CWKWJGRJQ06FHZ9MY86HE` review (blocker: "parallel editors for the same role"). The resolution chosen was the *wrong direction* — it kept the Combobox path and retired the CM6 path. This card reverses that: retire `ReferenceSelectEditor`, keep one unified CM6 path for all mention editors.

### Approach

1. **Create `SingleSelectEditor`** — new file `/Users/wballard/github/swissarmyhammer/swissarmyhammer-kanban/kanban-app/ui/src/components/fields/editors/single-select-editor.tsx`. Model: `multi-select-editor.tsx`. Differences:
   - Doc holds **at most one** mention token (`$my-project` or empty).
   - `parseDocTokens` returns 0 or 1 token — if multiple tokens somehow appear (paste, weird input), keep the **last resolved** token only (matches the prior `SelectReferenceEditor` behavior).
   - On commit: resolve to a scalar `string | null` (null when doc is empty). Never commits an array — the field's `type.multiple` is `false`.
   - Completion source: a `createSingleSelectCompletionSource` wrapper whose `apply` replaces the **whole doc** on accept (so autocomplete never appends — single-token invariant stays visual, not just enforced at commit).
   - Reuse `buildSubmitCancelExtensions` for Enter/Escape/blur semantics — matches `MultiSelectEditor`, no hand-rolled keymap branches.
   - Reuse `createMentionDecorations` so the selected token renders as a colored pill widget (same as multi-select).

2. **Register it** — new sibling `/Users/wballard/github/swissarmyhammer/swissarmyhammer-kanban/kanban-app/ui/src/components/fields/registrations/single-select.tsx` that calls `registerEditor("single-select", SingleSelectEditorAdapter)` — mirrors the one-registration-per-editor convention used by `multi-select.tsx`. Add its import to `.../registrations/index.ts`.

3. **Update the adapter routing** in `/Users/wballard/github/swissarmyhammer/swissarmyhammer-kanban/kanban-app/ui/src/components/fields/registrations/select.tsx`:
   - `SelectEditorAdapter` keeps dispatching static-options fields to `SelectEditor` (shadcn Select — unaffected).
   - For fields with `field.type.entity` set, delegate to `SingleSelectEditor` instead of `ReferenceSelectEditor`.
   - Update the docstring to match the final state.

4. **Delete the old editor + its tests**:
   - `/Users/wballard/github/swissarmyhammer/swissarmyhammer-kanban/kanban-app/ui/src/components/fields/editors/reference-select-editor.tsx`
   - `/Users/wballard/github/swissarmyhammer/swissarmyhammer-kanban/kanban-app/ui/src/components/fields/editors/reference-select-editor.test.tsx`
   - Remove `ReferenceSelectEditor` export from `/Users/wballard/github/swissarmyhammer/swissarmyhammer-kanban/kanban-app/ui/src/components/fields/editors/index.ts`.

5. **YAML is already correct**: `swissarmyhammer-kanban/builtin/definitions/project.yaml` (`editor: select`, `type.entity: project`, `display: badge`) and `.../position_column.yaml` (`editor: select`, `type.entity: column`) need no changes — the adapter-level migration handles both.

### Non-goals (explicit)

- Do NOT touch `SelectEditor` (the shadcn Select path for static-options fields like `position_ordinal`). Static enums stay on shadcn Select — they're not mention-based.
- Do NOT touch the Rust backend. `search_mentions` already works for `project` and `column`.
- Do NOT reintroduce the name `SelectReferenceEditor` (collision with the deleted file's git history would be confusing). Use `SingleSelectEditor` + registration key `"single-select"`.
- Do NOT extract shared helpers speculatively between `MultiSelectEditor` and `SingleSelectEditor`. If natural shared helpers fall out (a shared `resolveTokenToId`, for instance), extract them; otherwise keep parallel and simple.
