title: board-changed handler calls refresh() after already updating openBoards — double fetch of list_open_boards
---
kanban-app/ui/src/App.tsx:258-272\n\nThe `board-changed` listener manually fetches `list_open_boards`, sets state, then calls `refresh()`. But `refresh()` also calls `refreshBoards()` which fetches `list_open_boards` again. This means every `board-changed` event causes two `list_open_boards` IPC calls.\n\nThe two fetches can also produce inconsistent results: after the first fetch sets `openBoards`, `refresh()` runs and overwrites it with the second fetch result. If a rapid sequence of board-changed events fires, the state updates interleave unpredictably.\n\nSuggestion: Either (a) let `refresh()` handle the full update and remove the manual `list_open_boards` call from the listener, or (b) pass the already-fetched open list into `refresh()` to avoid re-fetching."
