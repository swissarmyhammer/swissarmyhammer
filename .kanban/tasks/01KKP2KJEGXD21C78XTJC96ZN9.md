builtin/skills/test-loop/SKILL.md:24-25\n\nStep 4 reads: \"If failure cards exist: run `/implement` to pick up and fix each one.\" The phrasing \"each one\" implies iterating over all cards, but `/implement` (by design) picks up exactly one card and stops. An agent following these instructions literally would run `/implement` once and then loop back to step 2 without knowing whether all failure cards have been processed.\n\nThis is a functional correctness concern: if there are 3 test-failure cards, a single `/implement` invocation fixes 1, and the loop might attempt to re-run `/test` before the remaining 2 are fixed, potentially creating duplicate cards on subsequent iterations.\n\nSuggestion: Change step 4 to: \"If failure cards exist: run `/implement` once (it picks up one card). Then return to step 2 to re-run `/test` and check for remaining failures.\" The current step 5 (\"Loop: go back to step 2\") partially covers this but is ambiguous about when to go back."
