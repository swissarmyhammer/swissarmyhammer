## What

The CM6 mention decorations (used in the filter formula bar and text editors)
render `#tag` mentions with different visual styling than the `MentionPill`
React component used in badge lists, inspector, and markdown display. They
should look identical.

### Current styling comparison

**MentionPill** (`kanban-app/ui/src/components/mention-pill.tsx`, `MentionPillInner`):
```css
backgroundColor: color-mix(in srgb, #${color} 20%, transparent)
color: #${color}
border: 1px solid color-mix(in srgb, #${color} 30%, transparent)
border-radius: rounded-full  (9999px)
padding: 0 6px 1px          (px-1.5 py-px)
font-size: 0.75rem           (text-xs)
font-weight: 500             (font-medium)
display: inline-flex
```

**CM6 decoration** (`kanban-app/ui/src/lib/cm-mention-decorations.ts`, `theme`):
```css
backgroundColor: color-mix(in srgb, var(--tag-color, #888) 20%, transparent)
color: var(--tag-color, #888)
border: MISSING
border-radius: 3px           (vs 9999px)
padding: 0 3px               (vs 0 6px 1px)
font-size: not set            (inherits from editor)
font-weight: 500
```

### Differences

1. **No border** — pills have `1px solid` at 30% opacity; CM6 decorations have none
2. **Wrong border-radius** — `3px` vs `9999px` (fully rounded pill shape)
3. **Smaller padding** — `0 3px` vs `0 6px 1px`
4. **No font-size** — CM6 inherits from editor; pills use `text-xs` (0.75rem)
5. **No `line-height`** — pills have it implicitly from `inline-flex items-center`

### Fix

Update the `theme` object in `createMentionDecorations` in
`cm-mention-decorations.ts` to match the `MentionPillInner` inline styles
exactly. The CSS custom property approach (`var(--tag-color)`) is fine — just
add the missing `border`, fix `borderRadius`, and adjust `padding`/`fontSize`.

### File to modify

- `kanban-app/ui/src/lib/cm-mention-decorations.ts` — update the `theme`
  object in `createMentionDecorations` (the `EditorView.baseTheme` block)
