## What

After card 01KMZZZ9AT adds the `window:*` moniker at the root of the scope chain, the separate `window_label` parameter on `dispatch_command` becomes redundant. Remove it ‚Äî the backend should derive window identity solely from the scope chain.

### Current state (after card 1)
- Scope chains include `window:{label}` as the root moniker
- `dispatch_command` still accepts `window_label: Option<String>` as a separate Tauri parameter (commands.rs:843)
- `dispatch_command_internal` extracts `window:*` from the scope chain BUT also accepts `window_label` as fallback (commands.rs:882)
- `backendDispatch()` helper still sends `windowLabel` alongside the scope chain (belt-and-suspenders)
- 6 command impls read `ctx.window_label` with `.unwrap_or(\"main\")` fallback

### Changes

**Backend ‚Äî `CommandContext`** (`swissarmyhammer-commands/src/context.rs`):
- Add `pub fn window_label_from_scope(&self) -> Option<&str>` ‚Äî scans `self.scope` for a `window:*` moniker, returns the label portion
- Change all 6 command impls to use `ctx.window_label_from_scope().unwrap_or(\"main\")` instead of `ctx.window_label.as_deref().unwrap_or(\"main\")`:
  - `ui_commands.rs:46` (InspectCmd)
  - `ui_commands.rs:69` (InspectorCloseCmd)
  - `ui_commands.rs:92` (InspectorCloseAllCmd)
  - `ui_commands.rs:190` (SetActiveViewCmd)
  - `file_commands.rs:134` (CloseBoardCmd)
  - `app_commands.rs:172` (DismissCmd)

**Backend ‚Äî `dispatch_command_internal`** (`kanban-app/src/commands.rs`):
- Remove `window_label` parameter from `dispatch_command_internal` signature (line 882)
- Remove `ctx = ctx.with_window_label(label.clone())` (lines 970-972)
- The scope chain already carries the window moniker; `window_label_from_scope()` reads it directly

**Backend ‚Äî `dispatch_command`** (Tauri command, `kanban-app/src/commands.rs:835`):
- Remove `window_label: Option<String>` from the `#[tauri::command]` signature (line 843)
- Remove it from the `dispatch_command_internal` call (line 853)

**Backend ‚Äî `menu.rs`**:
- `trigger_open_board` and other menu-originated dispatches: these don't have a scope chain from the frontend. They must build a synthetic scope chain with `[\"window:{label}\"]` from the menu event's window label, and pass it as `scope_chain`.

**Frontend ‚Äî `backendDispatch()`** (`command-scope.tsx`):
- Remove `windowLabel` from the invoke args ‚Äî scope chain is the only mechanism now
- Keep injecting `windowLabel` into the scope chain if not already present (for calls that don't go through the focus system)

**Cleanup ‚Äî `CommandContext`** (`swissarmyhammer-commands/src/context.rs`):
- Remove `window_label` field (line 23)
- Remove `with_window_label()` builder (lines 64-67)
- Remove associated tests (`with_window_label_sets_field`, `chaining_with_window_label_and_with_ui_state`)
- Add new tests for `window_label_from_scope()`
