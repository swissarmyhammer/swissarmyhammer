## What\nThe `LspDaemon` currently consumes stdin/stdout during the initialize handshake and holds them privately in `child: Option<Child>`. Nobody can send JSON-RPC requests through the running rust-analyzer process.\n\nRefactor so the daemon:\n1. Spawns the process and does the initialize/initialized handshake (as today)\n2. After successful handshake, creates an `LspJsonRpcClient` from the initialized pipes\n3. Stores the client and exposes it via `pub fn client(&mut self) -> Option<&mut LspJsonRpcClient>`\n4. On health check failure / restart, drops the old client and creates a new one after re-handshake\n5. The `Child` handle stays in the daemon for `try_wait()` health checks and `kill()` shutdown\n\nThe `LspJsonRpcClient` already exists in `swissarmyhammer-code-context/src/lsp_communication.rs` — it takes a `Child` and uses `stdin.as_mut()` / `stdout.as_mut()`. The refactor is about when and where that client gets created.\n\nNote: `LspJsonRpcClient::new()` currently takes ownership of a `Child`. It needs to be refactored to take `ChildStdin` + `ChildStdout` instead, since the daemon keeps the `Child` for lifecycle management.\n\nFiles:\n- `swissarmyhammer-lsp/src/daemon.rs` — store client, expose after handshake\n- `swissarmyhammer-code-context/src/lsp_communication.rs` — accept stdin/stdout instead of Child\n\n## Acceptance Criteria\n- [ ] `LspJsonRpcClient::new()` takes `ChildStdin` + `ChildStdout` (not `Child`)\n- [ ] `LspDaemon` creates client after successful handshake\n- [ ] `LspDaemon::client()` returns `Option<&mut LspJsonRpcClient>` (None if not running)\n- [ ] Client is dropped and recreated on daemon restart\n- [ ] Health checks still work (daemon keeps Child handle)\n- [ ] Graceful shutdown still works\n\n## Tests\n- [ ] Unit test: `LspJsonRpcClient::new()` works with piped stdin/stdout\n- [ ] Existing daemon tests still pass (start, health check, shutdown)\n- [ ] Integration test: get client from daemon, send a request, get response (requires rust-analyzer, `#[ignore]`)"
