title: Tag right-click shows 'Inspect task' instead of 'Inspect tag'
---
When right-clicking a TagPill on an EntityCard, the context menu shows \"Inspect task\" (from the parent EntityCard's FocusScope) instead of \"Inspect tag\" (from the TagPill's own FocusScope).\n\nRoot cause: The TagPill only registers `entity.inspect` when the tag entity is resolved (`tagMoniker` is truthy). If for any reason the tag lookup fails, the command isn't registered and the parent EntityCard's `entity.inspect` leaks through the scope chain.\n\nAdditionally, event propagation may be letting the parent FocusScope's context menu handler fire.\n\nFiles: `ui/src/components/tag-pill.tsx`, `ui/src/components/focus-scope.tsx`\n\n- [ ] Always register `entity.inspect` in TagPill scope (use `available: false` when tag not resolved) to block parent scope leakage\n- [ ] Add `e.stopPropagation()` in FocusScopeInner's `handleContextMenu` before delegating to useContextMenu as belt-and-suspenders\n- [ ] Verify scope chain shadowing works correctly with nested FocusScopes\n- [ ] Test: right-click tag shows 'Inspect tag', not 'Inspect task'\n- [ ] Run tests to verify no regressions #bug
