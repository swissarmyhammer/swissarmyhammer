## What

Add infrastructure artifacts for the MCP server runtime.

### 1. `kanban-cli/src/logging.rs` — file-based tracing

Create `logging.rs` with `FileWriterGuard` (flush+sync on every write) and a tracing init function, matching `swissarmyhammer-cli/src/logging.rs`. Log to `.kanban/mcp.log` when `.kanban/` exists, fall back to stderr otherwise. Do NOT auto-create `.kanban/` (it holds committed board data).

`main.rs` calls the init function from `logging.rs` — no tracing setup inline in main.

### 2. `.kanban/.gitignore` entry for `mcp.log`

Append `mcp.log` to `.kanban/.gitignore` if not already present.

### 3. `kanban-cli/.mcp.json` for local dev

```json
{
  "mcpServers": {
    "kanban": {
      "command": "kanban",
      "args": ["serve"]
    }
  }
}
```

Note: do NOT add `mod serve; mod registry; mod doctor;` to main.rs — that's the wiring card's job. Only add `mod logging;` and call the init function.

## Review Findings (2026-04-12 10:45)

### Nits
- [x] `kanban-cli/src/logging.rs` (tests module) — The `file_writer_guard_flushes_on_write` test exercises `swissarmyhammer_common::logging::FileWriterGuard`, which is already covered by `write_flushes_to_disk` in `swissarmyhammer-common/src/logging.rs`. Since this crate imports `FileWriterGuard` rather than redefining it, the local test duplicates upstream coverage and doesn't exercise anything this module owns. Consider replacing it with a test for `init_tracing` — e.g., a test (with `#[serial]` to avoid clobbering the global subscriber) that confirms `.kanban/` is not auto-created when absent, and that `.kanban/mcp.log` is created when the directory exists.
- [x] `kanban-cli/src/logging.rs` `init_tracing` — When `.kanban/` exists but `File::create` fails (permissions, disk full, read-only fs), the function silently falls back to stderr. The `sah-cli` analogue emits a `eprintln!("Warning: Could not setup MCP logging: {}. Falling back to stderr.", e)` in that case. Consider matching that behavior so users aren't surprised when logs don't appear in the expected file.
- [x] `kanban-cli/src/main.rs` — `logging::init_tracing(false)` hardcodes `debug = false`. The `debug` parameter is never exercised at any call site. This is acceptable as a placeholder since future cards will wire a CLI flag, but consider adding a `// TODO:` comment noting that a `--debug` / `-v` flag is pending so the wiring isn't forgotten.
