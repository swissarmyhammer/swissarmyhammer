**Part 2a of the YAML/MD storage conversion plan.**

Convert task files from `{id}.json` to `{id}.md` with YAML frontmatter for metadata and markdown body for description.

**Add serde_yaml dependency:** Add `serde_yaml = { workspace = true }` to `swissarmyhammer-kanban/Cargo.toml` (already in workspace at v0.9).

**TaskMeta helper struct:** Create a struct with all Task fields except `description` and `id`, used only for YAML frontmatter serialization. Include legacy field handling (`_legacy_tags`, `_legacy_subtasks`).

**context.rs changes:**
- `task_path()` → return `tasks/{id}.md` instead of `.json`
- `write_task()` → serialize TaskMeta as YAML frontmatter, append description as markdown body
- `read_task()` → try `.md` first, fall back to `.json` for backward compat
- `list_task_ids()` → accept both `.md` and `.json` extensions
- Add `parse_task_markdown()` helper to split frontmatter from body

**Example output format:**
```
---
title: Fix the login bug
position:
  column: doing
  ordinal: a0
---
Description text here with #tags
```
