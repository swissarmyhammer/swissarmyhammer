## What\n\nThe `Display` enum in `swissarmyhammer-fields/src/types.rs:79` is a closed enum that must be manually synced with YAML field definitions. Adding a new display type (like `progress-ring`) requires a Rust code change. This defeats the metadata-driven design — the frontend treats display as a string and dispatches to a registry, the backend should too.\n\nWorse, when a YAML field definition uses an unknown display value, serde deserialization silently fails and the entire field definition vanishes. No error, no warning. This caused `percent_complete` to disappear when `display: progress-ring` was added to the YAML.\n\n### Fix\n\n1. **Change `Display` to `String`** in `swissarmyhammer-fields/src/types.rs`. Remove the enum, use `pub display: Option<String>`. The `effective_display()` method can still provide defaults based on field type, returning a string.\n2. **Same for `Editor`** if it has the same problem — check if it's also a closed enum.\n3. **Fail loudly on YAML parse errors** in `FieldsContext::from_yaml_sources` (`swissarmyhammer-fields/src/context.rs`). When a field definition YAML fails to deserialize, log a warning with the field name and the parse error. Don't silently drop it.\n\n### Files to modify\n- `swissarmyhammer-fields/src/types.rs` — `Display` enum → String, `Editor` enum → String if applicable\n- `swissarmyhammer-fields/src/context.rs` — log warnings on YAML parse failures in `from_yaml_sources`\n- Update any Rust code that pattern-matches on `Display` or `Editor` variants\n\n## Acceptance Criteria\n- [ ] `display` field in YAML accepts any string without Rust code changes\n- [ ] YAML parse errors produce a visible warning with the field name and error\n- [ ] Adding a new display type only requires frontend registration, not backend enum changes\n- [ ] Existing display types continue to work\n\n## Tests\n- [ ] Rust unit test: parse a field YAML with `display: some-new-type` — succeeds, field loads\n- [ ] Rust unit test: malformed YAML produces a warning, doesn't crash\n- [ ] Run: `cd swissarmyhammer-fields && cargo test`\n- [ ] Run: `cd swissarmyhammer-kanban && cargo test`"
