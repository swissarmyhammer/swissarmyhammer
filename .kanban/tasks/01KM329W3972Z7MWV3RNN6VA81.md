## What
Restructure `AppConfig` to store all per-window state in a single `windows: HashMap&lt;String, WindowState&gt;` map. Remove the global fields that should be per-window.

**Remove from AppConfig:**
- `active_view_id: Option&lt;String&gt;` (global → per-window)
- `inspector_stack: Vec&lt;String&gt;` (global → per-window)
- `main_window: Option&lt;WindowGeometry&gt;` (special case → just `windows["main"]`)

**Rename/expand WindowBoardEntry → WindowState:**
```rust
pub struct WindowState {
    pub board_path: PathBuf,
    pub active_view_id: Option&lt;String&gt;,
    pub inspector_stack: Vec&lt;String&gt;,
    // geometry fields (all optional, as before)
    pub x: Option&lt;i32&gt;,
    pub y: Option&lt;i32&gt;,
    pub width: Option&lt;u32&gt;,
    pub height: Option&lt;u32&gt;,
    pub maximized: bool,
}
```

**Resulting AppConfig:**
```rust
pub struct AppConfig {
    pub recent_boards: Vec&lt;RecentBoard&gt;,
    pub keymap_mode: String,
    pub open_boards: Vec&lt;PathBuf&gt;,
    pub windows: HashMap&lt;String, WindowState&gt;,
}
```

**Files:**
- `kanban-app/src/state.rs` — struct definitions, remove `WindowGeometry` (folded into `WindowState`), update `AppConfig::default()`
- `kanban-app/src/state.rs` tests — update all `window_boards` references to `windows`, update geometry tests to use new shape

**Migration:** The load path (from card 1) should handle the old shape — if `window_boards` key exists in YAML but `windows` doesn't, map it. If `main_window`/`active_view_id`/`inspector_stack` exist at top level, migrate them into `windows["main"]`.

## Acceptance Criteria
- [ ] `AppConfig` has a single `windows` field, no `window_boards`, `main_window`, `active_view_id`, or `inspector_stack` at top level
- [ ] `WindowBoardEntry` is renamed to `WindowState` with view + inspector + geometry fields
- [ ] Old configs with `window_boards`/`main_window` migrate cleanly on load
- [ ] Main window is just `windows["main"]`, no special case
