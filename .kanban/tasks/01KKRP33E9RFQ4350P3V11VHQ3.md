```rust
if front.to_string_lossy().len() > 90 {
    let front = PathBuf::from(format!("/tmp/{}-{}-f.sock", prefix, short_hash));
    ...
}
```

When the computed path exceeds 90 characters the code silently falls back to `/tmp`. Two different prefix+workspace combinations could collide in `/tmp` if their 8-char hash suffixes match (birthday problem is meaningful for 8 hex chars = 2^32 space). Additionally, the fallback uses a hardcoded `/tmp` which is insecure on shared multi-user systems and will not work on systems without `/tmp` (e.g., some container images).

Suggestion: emit a `tracing::warn!` when the fallback is triggered. Consider using `std::env::temp_dir()` instead of the `/tmp` literal. Document the collision risk in the function doc comment. #review-finding
