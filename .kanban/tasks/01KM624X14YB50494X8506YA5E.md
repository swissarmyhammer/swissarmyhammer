## What

Register `entity.archive` and `entity.unarchive` as commands in the unified command architecture. These are generic entity-level commands (like `entity.delete`), not task-specific.

### YAML command definitions

Add to `swissarmyhammer-commands/builtin/commands/entity.yaml`:

```yaml
- id: entity.archive
  name: Archive
  scope: "entity:task"
  undoable: true
  context_menu: true
  keys:
    vim: dd
  params:
    - name: moniker
      from: target

- id: entity.unarchive
  name: Unarchive
  scope: "entity:task"
  undoable: true
  context_menu: true
  params:
    - name: moniker
      from: target
```

### Command trait implementations

Add to `swissarmyhammer-kanban/src/commands/entity_commands.rs`:

- `ArchiveEntityCmd` ‚Äî parses moniker from target, calls `ectx.archive(entity_type, id)`. Available when target moniker is set.
- `UnarchiveEntityCmd` ‚Äî parses moniker from target, calls `ectx.unarchive(entity_type, id)`. Available when target moniker is set.

### Registration

Add to `swissarmyhammer-kanban/src/commands/mod.rs` `register_commands()`:
- `"entity.archive"` ‚Üí `ArchiveEntityCmd`
- `"entity.unarchive"` ‚Üí `UnarchiveEntityCmd`

Update the command count assertion in the test.

### Keybinding change

In `kanban-app/ui/src/lib/keybindings.ts`, the vim `dd` sequence already maps to `task.archive`. Change it to `entity.archive` since the command is now entity-generic. The scope chain will resolve the correct entity type.

### Files
- `swissarmyhammer-commands/builtin/commands/entity.yaml` ‚Äî add command defs
