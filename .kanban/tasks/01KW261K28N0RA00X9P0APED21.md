## What
Discover and load expectations, and resolve a `<scope>` argument to a set of them. Template: `crates/swissarmyhammer-validators/src/validators/loader.rs`.

- New `crates/swissarmyhammer-expect/src/loader.rs`:
  - `ExpectationLoader` that discovers `**/*.expect.md` across the repo (colocated specs anywhere) plus repo-global specs under `.expect/expectations/`. Skip the usual ignored dirs (reuse the skip-dir set used by code_context/project-detection).
  - `load_all(repo_root) -> Result<Vec<Expectation>, ExpectError>`.
  - `resolve_scope(scope: &str | None, repo_root, tag: Option<&str>) -> Vec<Expectation>` implementing the three forms from `ideas/expect.md` §"Scope resolution":
    1. a specific expectation — path to a `*.expect.md`, or its repo-relative path with extension dropped (`src/checkout/coupon`);
    2. a folder — every `*.expect.md` under it, recursively;
    3. a glob — shell-style (`src/**/*pricing*.expect.md`) or `--tag <name>`.
    With no scope: discover every `**/*.expect.md` (the default CI invocation).
- Identity is the repo-relative path (ext dropped); dedupe by it.

## Acceptance Criteria
- [ ] Given a temp repo with specs at `src/a.expect.md`, `src/checkout/coupon.expect.md`, `.expect/expectations/global.expect.md`: `load_all` finds all three.
- [ ] `resolve_scope("src/checkout/coupon")` ⇒ exactly the coupon spec (extension-dropped form).
- [ ] `resolve_scope("src/checkout/")` ⇒ all specs under that folder.
- [ ] `resolve_scope("src/**/*coupon*.expect.md")` (glob) and `--tag` selection both work; empty scope ⇒ all specs.

## Tests
- [ ] `crates/swissarmyhammer-expect/src/loader.rs` unit tests using a `tempfile` repo fixture for each scope form (path, ext-dropped, folder, glob, tag, none).
- [ ] `cargo nextest run -p swissarmyhammer-expect loader` passes.
