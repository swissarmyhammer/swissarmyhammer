title: 'Migrate React command dispatch: scope chain, set_focus, dispatch_command'
---
Rewrite React's command system to dispatch through the new Rust `dispatch_command` endpoint. React computes scope chains and sends focus updates.

## Scope

- Replace all `invoke("execute_command", ...)` calls with `invoke("dispatch_command", ...)`
- Implement `set_focus` call on every focus change — build scope chain by walking FocusScope parent chain, send `Vec<String>` of monikers to Rust
- Rewrite `AppShell` global commands: remove JS `execute` closures, use `dispatch_command` with command IDs only
  - `app.undo` → `dispatch_command({ cmd: "app.undo" })`
  - `app.palette` → `dispatch_command({ cmd: "ui.palette.open" })`
  - `settings.keymap.*` → `dispatch_command({ cmd: "settings.keymap.vim" })` etc.
- Rewrite `EntityCard`, `ColumnView`, `BoardView` FocusScope commands: remove `execute` closures, commands just have `id` and React dispatches on activation
- Rewrite `TagPill` commands: remove `rustCommand` field (no longer needed), just dispatch by ID
- Rewrite `BoardView` drag-and-drop: dispatch `task.move` with `target: "column:XXX"` and `args: { drop_index }` instead of computing ordinal in React
- Rewrite context menu: call `list_available_commands({ context_menu: true })` from Rust instead of collecting from scope chain in React
- Remove `CommandDef.execute` and `CommandDef.rustCommand` fields — all commands dispatch to Rust
- Keep `CommandScopeProvider` for scope chain building but simplify — it no longer needs command maps, just moniker chain
