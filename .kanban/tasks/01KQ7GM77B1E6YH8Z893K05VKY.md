## What

Double-clicking a perspective tab opens the inspector. It should not. **Perspective is not an entity** ‚Äî perspectives are saved view-state (filter + group + sort) attached to a board, not first-class inspectable objects. The inspector exists to show details of entities (board, column, task, tag, field), so dispatching `ui.inspect` against a `perspective_tab:<id>` moniker is an architectural mismatch.

The user reports the symptom for the perspective tab, but the root cause is broader: every UI-chrome focusable currently inherits the same incorrect behavior.

### Why it's broken

`kanban-app/ui/src/components/focus-scope.tsx` (and the symmetric `focus-zone.tsx`) wires an unconditional double-click handler:

```tsx
const dispatch = useDispatchCommand("ui.inspect");
‚Ä¶
const handleDoubleClick = useCallback((e) => {
  // ‚Ä¶guards‚Ä¶
  e.stopPropagation();
  dispatch({ target: moniker }).catch(console.error);
}, [dispatch, moniker, handleEvents]);
```

So **every** `<FocusScope>` / `<FocusZone>` wrapper, regardless of what it represents, fires `ui.inspect` with its own moniker on double-click. The set of focusables today includes a lot of pure UI chrome (`perspective_tab:<id>`, `ui:navbar.*`, `ui:toolbar.*`, `ui:perspective-bar`, `cell:<row>:<col>`, etc.) that does not correspond to an inspectable entity.

The perspective tab additionally has its own `<button onDoubleClick={onDoubleClick}>` that calls `startRename(p.id)`. The button does not call `e.stopPropagation()`, so the double-click bubbles to the wrapping `<Focusable>` which fires `ui.inspect` ‚Äî both rename **and** inspect run from one user gesture. Visible result: the inspector opens.

### Fix shape ‚Äî `inspectOnDoubleClick` opt-in prop

Make inspect-on-double-click an **explicit opt-in** on `<FocusScope>` / `<FocusZone>` via a new prop `inspectOnDoubleClick?: boolean` defaulting to `false`. Only entity call sites opt in.

Why opt-in (not opt-out via prefix detection or scope chain inspection):

- The set of entity prefixes is owned by the kanban domain (`task:`, `tag:`, `column:`, `board:`, `field:`, `attachment:`). Hard-coding that list in a generic spatial primitive smears domain knowledge into UI infrastructure.
- A scope-based runtime check is brittle ‚Äî the answer depends on whichever `CommandScopeProvider` happens to be the nearest ancestor, not on what *this* focusable represents.
- Opt-in flips the contract from "inspect everything unless told otherwise" (the current bug) to "inspect only entity-bearing call sites that explicitly say so." That matches how the rest of the spatial primitives work (`showFocusBar`, `navOverride` ‚Äî explicit, not inferred).

Note: a follow-up task replaces this boolean with a dedicated `<Inspectable>` wrapper component for cleaner separation of concerns. This task ships the boolean as the urgent bug fix; the wrapper-based refactor follows.

## Implementation summary

Implemented `inspectOnDoubleClick` opt-in prop on `<FocusScope>` / `<FocusZone>` via a small inner `<InspectDoubleClickRegistrar>` child component pattern: when the prop is true, the registrar mounts and calls `useDispatchCommand("ui.inspect")`, installing the dispatcher into a ref that the body's `onDoubleClick` reads. When false (the default), the registrar does not mount, the hook is never called, and `onDoubleClick` is not attached to the body div at all ‚Äî so the gesture bubbles to ancestors untouched.

Entity call sites opted-in: `entity-card.tsx`, `column-view.tsx`, `board-view.tsx`, `fields/field.tsx`, `data-table.tsx` (entity-row scope), `mention-view.tsx` (mention pills), `fields/displays/attachment-display.tsx`. UI-chrome call sites stay opt-out by default (no code changes). The column-name navigation leaf in `column-view.tsx` carries `// inspect:exempt` because it wraps a synthetic `column:<id>.name` moniker whose inner `<Field>` zone owns the inspect opt-in.

Architectural guard added to `focus-architecture.guards.node.test.ts`. Two new test files added: `perspective-tab-bar.no-inspect-on-dblclick.spatial.test.tsx` (4 tests) and `focus-scope.inspect-opt-in.spatial.test.tsx` (11 tests). Updated 2 existing tests in `focus-scope.test.tsx` to add the prop now that it's required.
