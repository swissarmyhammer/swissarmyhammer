## What\n\nWhen dragging a card in a column with many tasks, there's no way to scroll to reach drop positions beyond the visible area. The column container (`overflow-y-auto` div in `column-view.tsx:180-189`) needs to auto-scroll when the drag cursor is near the top or bottom edge.\n\n**Fix**: In `handleDragOver` (`column-view.tsx`), after computing the insert index, check if `e.clientY` is within a threshold (e.g. 40px) of the container's top or bottom edge. If so, scroll the container in that direction using `containerRef.current.scrollBy()`. Use `requestAnimationFrame` to keep scrolling smooth while the cursor stays in the edge zone.\n\n### Approach\n- Define a scroll zone threshold (e.g. `SCROLL_ZONE = 40` px from each edge)\n- In `handleDragOver`, get `containerRef.current.getBoundingClientRect()`\n- If `clientY` is within `SCROLL_ZONE` of top edge → `scrollBy({ top: -SCROLL_SPEED })`\n- If `clientY` is within `SCROLL_ZONE` of bottom edge → `scrollBy({ top: SCROLL_SPEED })`\n- Use a `requestAnimationFrame` loop stored in a ref so scrolling continues while hovering in the zone\n- Clear the rAF on drag leave and drop\n\n### Files to modify\n- `kanban-app/ui/src/components/column-view.tsx` — add edge-scroll logic to `handleDragOver`, cleanup in `handleDragLeave` and `handleDrop`\n\n## Acceptance Criteria\n- [ ] Dragging near the bottom edge of a scrollable column scrolls down\n- [ ] Dragging near the top edge scrolls up\n- [ ] Scrolling stops when cursor moves away from the edge zone\n- [ ] Scrolling stops on drop and drag leave\n- [ ] Insert marker updates correctly while scrolling\n\n## Tests\n- [ ] Manual: create a column with 20+ tasks, drag a card to the bottom edge → column scrolls down\n- [ ] Manual: drag to top edge → column scrolls up\n- [ ] Run: `cd kanban-app/ui && npm test` — no regressions"
