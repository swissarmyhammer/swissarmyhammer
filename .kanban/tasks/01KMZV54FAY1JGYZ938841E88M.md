title: Open Board does not switch current window to show the board

## What

File > Open Board shows the folder picker, user picks a board folder, but the current window does not switch to show that board. The board is not loaded as active.

### Root cause
`OpenBoardCmd` returns `{ OpenBoardDialog: true }`. `dispatch_command_internal` calls `menu::trigger_open_board(app)` which triggers an async dialog. The dialog callback calls `open_and_notify` → `dispatch_command_internal(file.switchBoard)`. Potential deadlock in `update_menu_enabled_state` (`try_read` on boards lock).

### Files to investigate
- `kanban-app/src/commands.rs` lines 1009-1010 — OpenBoardDialog handler
- `kanban-app/src/menu.rs` — `open_and_notify` nested dispatch
