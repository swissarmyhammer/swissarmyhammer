## What\nImplement the kanban-cli entry point that is literally `sah tool kanban` as a standalone binary, plus an `open` command and banner.\n\n**File:** `kanban-cli/src/main.rs`\n\n**CLI structure:** Use the shared CLI generation utilities (extracted in prior card) to build the exact same noun-verb subcommand tree that `sah tool kanban` has:\n- `kanban board init --name \"Test\"`\n- `kanban task add --title \"Hello\"`\n- `kanban task move --id \"...\" --column doing`\n- `kanban column list`\n- etc.\n\n**Execution:** Use the shared argument extraction to build JSON args from clap matches, then call the kanban MCP tool's `execute()` directly (same as `sah tool kanban` does). No new dispatch code.\n\n**Open command:** `kanban .` or `kanban <path>` resolves path to absolute, URL-encodes, opens `kanban://open/<path>` via `open::that()`. Use `allow_external_subcommands(true)` — unrecognized subcommands that look like paths trigger open.\n\n**Banner:** Show on bare `kanban` or `kanban --help`.\n\n## Acceptance Criteria\n- [ ] `kanban board init --name \"Test\"` behaves identically to `sah tool kanban board init --name \"Test\"`\n- [ ] `kanban task add --title \"Hello\"` works\n- [ ] `kanban .` opens `kanban://open/...` URL\n- [ ] All noun-verb subcommands match `sah tool kanban` exactly\n\n## Tests\n- [ ] Integration test: init board, add task, list tasks via CLI\n- [ ] `cargo nextest run -p kanban-cli` passes
