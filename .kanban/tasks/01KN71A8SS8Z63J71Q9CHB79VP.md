## What

Right-clicking a tag pill in the **board card view** shows the full merged context menu (tag commands like "Remove Tag", task commands, column commands, board commands) because the scope chain walks: `tag:id → task:id → column:id → board`. But right-clicking a tag pill in the **inspector view** only shows tag-level commands because the scope chain is: `tag:id → task:id.tags (field row, empty commands) → CommandScopeProvider (nav commands only)`. The inspector never wraps content in an entity-level FocusScope for the inspected entity, so task/column/board commands are absent from the scope chain.

### Root Cause

`InspectorFocusBridge` (`kanban-app/ui/src/components/inspector-focus-bridge.tsx`) wraps `EntityInspector` in a `CommandScopeProvider` that only has inspector navigation commands. It does not register the inspected entity's commands (from `useEntityCommands`) in the scope chain. The `FieldRow` in `entity-inspector.tsx:327` has `commands={[]}`.

### Fix

In `inspector-focus-bridge.tsx`, wrap the `EntityInspector` in a `FocusScope` (not just `CommandScopeProvider`) using the entity's moniker and `useEntityCommands(entity.entity_type, entity.id, entity)`. This puts the inspected entity's commands (inspect, archive, etc.) into the scope chain so that any child pill's right-click context menu includes them — just like the board card view.

The existing `CommandScopeProvider` for nav commands can nest inside or outside the entity `FocusScope` — the scope chain merges both.

### Files to modify

- `kanban-app/ui/src/components/inspector-focus-bridge.tsx` — add `FocusScope` with entity commands wrapping `EntityInspector`

No code duplication introduced — both board card and inspector will use `useEntityCommands` to register entity commands. The board card already does this in `entity-card.tsx:84-89`.
