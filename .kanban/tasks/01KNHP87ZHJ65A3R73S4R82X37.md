**File:** `kanban-app/ui/src/components/column-view.tsx` — two sites\n\n**What:**\n- `moniker(\"column\", column.id)` for columnMoniker\n- `fieldMoniker(\"column\", column.id, \"name\")` for columnNameMoniker\n- `tasks.map((t) => moniker(\"task\", t.id))` for taskMonikers\n\nAll entity objects are available in scope (`column`, `tasks`).\n\n**Why:** Same pattern as board-view — these are focus navigation monikers. They work for non-archived entities but bypass the backend moniker. Consistent migration means using `entity.moniker` as the source of truth.\n\n**Suggestion:** Use `column.moniker` and `t.moniker` respectively. For `fieldMoniker`, refactor to accept a base moniker string + field name: `fieldMonikerFrom(column.moniker, \"name\")` or `column.moniker + \".name\"`.\n\n**Verification:** Column header editing and card keyboard navigation continue to work. #review-finding
