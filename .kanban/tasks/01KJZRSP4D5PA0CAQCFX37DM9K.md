Create a new generic, consumer-agnostic `swissarmyhammer-views` crate following the same pattern as `swissarmyhammer-fields` but with full CRUD and changelog.\n\nViews are **mutable** — users can create, modify, and delete them. Changes are logged with whole-view snapshot undo/redo.\n\n## Key files\n- `swissarmyhammer-views/Cargo.toml` — workspace crate\n- `swissarmyhammer-views/src/lib.rs` — pub mod types, context, changelog, error\n- `swissarmyhammer-views/src/types.rs` — ViewDef, ViewId, ViewKind, ViewCommandDef\n- `swissarmyhammer-views/src/context.rs` — ViewsContext with read/write/delete/list + from_yaml_sources()\n- `swissarmyhammer-views/src/changelog.rs` — ViewChangeEntry with whole-view snapshots, JSONL, undo/redo\n- `swissarmyhammer-views/src/error.rs` — ViewsError, Result type alias\n\n## ViewDef type\n```rust\nstruct ViewDef {\n    id: ViewId,                      // ULID\n    name: String,                    // Human-facing\n    icon: Option<String>,            // Lucide icon name\n    kind: ViewKind,                  // Grid | Board\n    entity_type: String,             // Which entity type this view shows\n    card_fields: Option<Vec<String>>, // Fields to show on cards (board-specific)\n    commands: Vec<ViewCommandDef>,   // Command metadata (data-driven)\n}\n\nstruct ViewCommandDef {\n    id: String,                      // Command ID (e.g. \"board.newCard\")\n    name: String,                    // Human-readable name\n    description: Option<String>,\n    keys: Option<KeyBindings>,       // Per-keymap keybindings\n}\n\nstruct KeyBindings {\n    vim: Option<String>,\n    cua: Option<String>,\n    emacs: Option<String>,\n}\n```\n\nCommands are data-driven: ViewDef declares command metadata (id, name, keys). React renderers attach execute functions at mount time by matching command id.\n\n## ViewsContext\n- `from_yaml_sources(writable_root, &[(&str, &str)])` — parse YAML, build name+id indexes\n- `get_view_by_name()`, `get_view_by_id()`, `all_views()`\n- `write_view(def)` — write YAML to disk, update indexes, append changelog (whole-view snapshot)\n- `delete_view(id)` — remove file, update indexes, append changelog\n- `undo(entry_id)` / `redo(entry_id)` — replay snapshots\n\n## Checklist\n- [ ] Create Cargo.toml with workspace deps\n- [ ] Add to workspace Cargo.toml members\n- [ ] Implement ViewDef, ViewId, ViewKind, ViewCommandDef, KeyBindings types with serde\n- [ ] Implement ViewsContext with from_yaml_sources() and CRUD\n- [ ] Implement ViewChangeEntry changelog with JSONL append/read\n- [ ] Implement undo/redo via snapshot replay\n- [ ] Implement error types\n- [ ] Write tests: YAML parsing, write/delete round-trip, changelog, undo/redo, override merge\n- [ ] cargo test -p swissarmyhammer-views passes\n\n#phase2-views
