## Avatar component design
- Props: `actorId: string`, `size?: "sm" | "md" | "lg"`, `className?`
- Uses `useEntityStore().getEntity("actor", actorId)` to resolve actor
- If actor has `avatar` field (data URI): render `<img>` with rounded-full
- Fallback: colored circle with initials (first letter of each word in name)
- Color from actor's `color` field or deterministic hash

## AvatarDisplay design (for assignees reference field)
- `value` is array of actor IDs
- Renders row of overlapping `<Avatar>` components
- Shows "+N" if more than 3
- Compact/full modes
