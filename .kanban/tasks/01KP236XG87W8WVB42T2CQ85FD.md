## What

Also: the `started` and `completed` field descriptions are currently long sentences. Shorten them so they read well as inline help text.

## Implementation notes
- The card's reference to `field.tsx:162` only partially held: `field.tsx#FieldEditor` does spread `field` / `entity` into `<Editor>`, but the registry-level adapters (`registrations/date.tsx`, `registrations/number.tsx`, `registrations/color-palette.tsx`, etc.) intercept and **strip** these before forwarding to the concrete editor. Extending `EditorProps` alone would type-check at the adapter boundary but leave `field` undefined at runtime in `DateEditor`. Fix: every editor adapter was updated to forward `field` (and `entity` where it was also missing) through to the concrete editor. All editors now receive the YAML field definition.
- Added a second `describe` block to `date-editor.test.tsx` covering empty-state rendering and CM6 placeholder — the existing unit tests in that file cover submit/cancel ref wiring in isolation, so I kept both concerns in a single file grouped by their own `describe`.
