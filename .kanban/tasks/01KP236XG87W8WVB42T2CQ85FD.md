## What

When a date field is unset, the UI currently renders a muted `-`. Instead, show the field's `description` (from the YAML def) as muted placeholder/help text. The same description should drive the date picker's inline placeholder when the editor opens.

Also: the `started` and `completed` field descriptions are currently long sentences. Shorten them so they read well as inline help text.

### Frontend changes

**File:** `/Users/wballard/github/swissarmyhammer/swissarmyhammer-kanban/kanban-app/ui/src/components/fields/editors/index.ts`

Extend `EditorProps` to expose the field definition:

```ts
import type { FieldDef, Entity } from "@/types/kanban";

export interface EditorProps {
  value: unknown;
  field: FieldDef;        // NEW — already passed from field.tsx:162
  entity: Entity;         // NEW — already passed from field.tsx:164 (confirm shape via FieldDisplayProps)
  ...
}
```

`field.tsx` already spreads `field={fieldDef}` and `entity={entity}` into the `<Editor>` (lines 161–169) — the interface just needs to reflect that. Confirm no other editor cares.

**File:** `/Users/wballard/github/swissarmyhammer/swissarmyhammer-kanban/kanban-app/ui/src/components/fields/displays/date-display.tsx`

- Accept `field` (already available via `DisplayProps`).
- When value is empty, render `field.description` in the existing muted style instead of `-`. Fall back to `-` if description is absent.

**File:** `/Users/wballard/github/swissarmyhammer/swissarmyhammer-kanban/kanban-app/ui/src/components/fields/editors/date-editor.tsx`

- Accept `field` via `EditorProps`.
- In the `PopoverTrigger` branch (currently renders `-` at the empty-value path), render `field.description` in the muted style when no value is set.
- Replace the hardcoded CodeMirror `placeholder="Type a date... (e.g. tomorrow, next friday)"` with `field.description ?? "Type a date... (e.g. tomorrow, next friday)"` — keep the current string as fallback for fields without descriptions.
