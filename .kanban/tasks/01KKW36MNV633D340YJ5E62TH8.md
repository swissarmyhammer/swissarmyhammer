title: board.switch.<path> command ID contains raw filesystem path — breaks command registry lookup
---
kanban-app/ui/src/components/app-shell.tsx:268\n\nThe dynamic board switch command is registered with id `board.switch.${b.path}`, where `b.path` is a raw filesystem path such as `/Users/alice/projects/myboard/.kanban`. This produces an id like `board.switch./Users/alice/projects/myboard/.kanban` which:\n1. Violates the `dispatch_command` validation on the Rust side that requires command IDs to be ASCII with max 128 chars — many paths will exceed 128 characters or could contain non-ASCII characters, causing `dispatch_command` to reject it.\n2. Contains `/` and `.` characters that may cause issues when the id is used as a CSS selector or data-testid attribute.\n\nThese commands use a local `execute` handler (not routed to Rust), so the 128-char/ASCII check in `dispatch_command` isn't hit at runtime, but it's still fragile. A path with non-ASCII folder names (e.g. emoji in directory names) would produce a non-ASCII command id.\n\nSuggestion: Use an index or hash as the id suffix, e.g. `board.switch.${index}` or `board.switch.${btoa(b.path).slice(0, 16)}`, keeping the human-readable name in the `name` field."
