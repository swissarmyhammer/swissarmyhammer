builtin/skills/test-loop/SKILL.md:3\n\nThe frontmatter description says \"continuously run tests, create failure cards, and implement fixes\". The phrase \"implement fixes\" overstates what the skill controls — it delegates to `/implement` which picks up whatever card is next, not specifically fix cards. This works in practice (because the only cards on the board should be test-failure cards in this context), but the description could mislead users who have a mixed board.\n\nSuggestion: Change to \"continuously run tests, create failure cards, and fix failures until the suite is fully green\"." #review-finding
