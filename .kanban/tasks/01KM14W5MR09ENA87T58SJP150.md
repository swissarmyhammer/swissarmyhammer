## What

Extend the existing `sah tools` subcommand (under `Commands::Serve` area or as a new top-level match arm in `route_subcommand()`) with enable/disable verbs. Do NOT touch `sah tool` — that's for tool execution.

**Syntax:**
- `sah tools` — list all tools with enabled/disabled status
- `sah tools enable` — enable all tools
- `sah tools disable` — disable all tools
- `sah tools enable shell git` — enable specific tools (variadic names)
- `sah tools disable kanban web` — disable specific tools (variadic names)
- All commands accept `--global` to write to `~/.sah/tools.yaml` instead of `.sah/tools.yaml`

**Implementation:**
- Add `Some(("tools", sub_matches))` arm to `route_subcommand()` in `swissarmyhammer-cli/src/main.rs` (before the catch-all)
- Add `Tools` variant to `Commands` enum in `cli.rs` with subcommands: `Enable { names: Vec<String> }`, `Disable { names: Vec<String> }`, and default (no subcommand = list)
- New: `swissarmyhammer-cli/src/commands/tools/mod.rs` — handle_command for enable/disable/list
- The commands: read current tools.yaml → mutate → write back → print status table

**UX for `sah tools`:**
```
TOOL           STATUS
agent          enabled
code_context   enabled
files          enabled
git            enabled
kanban         disabled
question       enabled
ralph          enabled
shell          enabled
skill          enabled
web            disabled
```

**UX for `sah tools disable`:**
```
All tools disabled.
```

**UX for `sah tools enable shell`:**
```
shell: enabled
```

**Files:**
- `swissarmyhammer-cli/src/cli.rs` — add Tools subcommand with Enable/Disable/list
- New: `swissarmyhammer-cli/src/commands/tools/mod.rs` — command handler
- `swissarmyhammer-cli/src/main.rs` — add match arm in route_subcommand()
