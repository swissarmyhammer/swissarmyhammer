## What

The new `arguments` field threading (from MCP JSON input → `render_skill_instructions()` → `{{arguments}}` in Liquid template) has no automated test. The existing tests in `swissarmyhammer-tools/src/mcp/tools/skill/mod.rs` and `swissarmyhammer-tools/tests/integration/skill_e2e.rs` exercise `use skill` but none pass an `arguments` value and verify it appears in the rendered output.

Without this test, a regression where arguments are silently dropped would go undetected.

Files:
- `swissarmyhammer-tools/src/mcp/tools/skill/mod.rs` (tests section, ~line 274)
- `swissarmyhammer-tools/tests/integration/skill_e2e.rs`

## Acceptance Criteria
- [ ] A unit test exists that calls `execute_use()` (or the SkillTool.execute wrapper) with `{"op": "use skill", "name": "card", "arguments": "fix the login bug"}` and asserts the rendered instructions contain `"fix the login bug"`.
- [ ] An e2e/integration test in `skill_e2e.rs` does the same over the full MCP stack.

## Tests
- [ ] `cargo test -p swissarmyhammer-tools -- test_use_skill_with_arguments` passes.
- [ ] `cargo test -p swissarmyhammer-tools --test integration -- skill_e2e` passes. #review-finding #warning
