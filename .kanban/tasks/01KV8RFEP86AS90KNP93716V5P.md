## What
The `search tasks` op (`crates/swissarmyhammer-kanban/src/task/search.rs`) currently maps each ranked `Hit` back to the FULL enriched task JSON via `task_entity_to_rich_json`, then attaches `score`/`signals`. The "slim map-back" revision on card `bchrrpg` (01KTC9VVAW145DZ7626BCHRRPG) supersedes this: `search tasks` results must use the SAME slim projection as `list tasks` (`slim_task_json` / `TaskDetail::Slim`) â€” an allowlist projection that EXCLUDES description/comments/attachments â€” plus `score`/`signals`. The agent follows up with `get task` (always full) on the hit it cares about. This keeps AI tool-call payloads lean (heavy enriched JSON Ă— up to top_k hits is a large token cost).

The slim infra (`task_helpers::slim_task_json`, `task/shared.rs::TaskDetail::Slim` + `parse_detail`) already exists on main and is used by `list tasks`/`add`/`paste`.
