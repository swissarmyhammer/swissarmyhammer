**File:** `kanban-app/ui/src/lib/entity-commands.ts` — `buildEntityCommandDefs()` and `useEntityCommands()`\n\n**What:** Both functions construct the entity moniker via `moniker(entityType, entityId)` from separate arguments rather than using `entity.moniker` when an entity is available. The entity parameter is optional and used only for template resolution, not for moniker.\n\n**Why:** These functions receive `entityType` and `entityId` as separate parameters and don't have guaranteed access to the Entity object. However, for the callers that DO pass an entity, the moniker could be read from `entity.moniker` directly — which would correctly handle archive monikers (e.g. `task:01ABC:archive`) that simple `type:id` construction misses.\n\n**Suggestion:** When `entity` is provided and has a truthy `moniker` property, use `entity.moniker` as the target. Fall back to `moniker(entityType, entityId)` when entity is not provided. This covers the archive moniker case.\n\n**Verification:** Inspect an archived entity and confirm the command target moniker includes the `:archive` suffix. #review-finding
