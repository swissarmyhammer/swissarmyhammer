## Symptom
A `review sha` over a large/multi-commit range fails ALL fan-out tasks (e.g. 15/15), tripping the completeness guard (`check_review_completeness`, >50% failed) and returning an error. Small/single-commit reviews (what `/finish` runs as `HEAD~1..HEAD`) are fine. The engine's error text blames "saturated/unavailable agent queue" — that is a guess and is WRONG.

## Root cause (verified by code + reproduction)
The fan-out grain is the VALIDATOR, not (validator × file). All changed files are inlined ONCE into a single shared "primed prefix" (`fleet.rs::render_run_prime`), and each of ~M validators forks that prime and runs its whole rule-set over ALL files at once. So it is **M invocations, each crammed with every diff** — NOT N×M.

There is **no aggregate size budget on the prime**. When the combined content exceeds the review model's context window the prime turn fails; the per-task fallback ALSO carries everything and also fails → every validator fails uniformly and immediately. Hard cliff, all-or-nothing.

## Reproduction
`attempted` is constant 15 regardless of file count — proof the grain is per-validator (M), not N×M. 75 files (mostly deletions) PASSED but 114-files/~974-insertions FAILED → it is aggregate inlined content volume, not file count.

## Decided design — content-budgeted batching at file granularity
- **`batch_size` parameter on the review tool**, default **32 KB (32768 bytes)**. NOT model/context-window-driven — deliberately small (cramming reviews poorly even when it fits).
- **Whole-file granularity.** Greedily pack whole files into a batch until adding the next would exceed `batch_size`; then start a new batch. **NEVER split a file across batches.**
- **A single file whose inlined content exceeds `batch_size` is a HARD ERROR** (not sliced, not spilled) naming the file/size/limit.
- Each batch runs the full fan-out (own prime, fork per validator); findings merged + deduped by file:line across batches.
- Replaces the silent `MAX_INLINE_SOURCE_BYTES` slice-degrade with the explicit error.

## Acceptance
- A 114-file-class range completes (batched) instead of 15/15 failing. Small reviews unchanged (1 batch). A single file > `batch_size` errors clearly; no file is ever split. `batch_size` settable per call, default 32 KB.
