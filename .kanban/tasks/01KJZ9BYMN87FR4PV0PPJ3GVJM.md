Create a FocusScope component that manages entity focus (which entity is selected for action) independently from DOM input focus.\n\n**FocusScope component:**\n```tsx\ninterface FocusScopeProps {\n  moniker: string;\n  commands: CommandDef[];\n  children: ReactNode;\n  as?: keyof JSX.IntrinsicElements; // default \"div\"\n}\n```\n\nFocusScope wraps its children in:\n1. `CommandScopeProvider` with the given commands\n2. A DOM element that:\n   - Sets entity focus on click and contextmenu (not on input focus)\n   - Attaches `onContextMenu={useContextMenu()}`\n   - Adds `data-focused` attribute when this moniker matches the focused entity\n   - Adds `data-moniker` attribute for debugging/CSS\n\n**EntityFocusContext:**\n```tsx\ninterface EntityFocusContextValue {\n  focusedMoniker: string | null;\n  setFocus: (moniker: string | null) => void;\n}\n```\n\nProvided once at the App level. Any FocusScope reads from this context and writes to it on click.\n\n**Key behaviors:**\n- Click on a FocusScope sets entity focus to its moniker\n- Right-click on a FocusScope sets entity focus AND shows context menu\n- Clicking an input/textarea inside a FocusScope does NOT change entity focus (only the outermost FocusScope receives the click)\n- `useFocusedEntity()` hook returns the current moniker — usable by command palette, vim commands, status bar\n- CSS can target `[data-focused]` for visual ring/highlight\n- Entity focus is cleared when clicking the backdrop or empty board area\n\n**Interaction with input focus:**\n- FocusScope does NOT call `element.focus()` or set tabIndex\n- Input focus is managed by individual input components as usual\n- Entity focus and input
