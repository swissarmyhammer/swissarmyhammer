## What\nThere is no LSP indexing loop. The `LspDaemon` runs rust-analyzer and the `LspJsonRpcClient` can send requests, but nobody wires them together in a worker that walks dirty files.\n\nThe daemon IS the managed server. The indexing worker is a client of it — it reuses the daemon's stdin/stdout pipes to send JSON-RPC requests. The daemon already did the initialize handshake, so the worker just needs to send `textDocument/documentSymbol` requests through the existing connection.\n\n**Design**: Add a method to `LspDaemon` that exposes a JSON-RPC send/receive interface (or hands back mutable access to stdin/stdout). The indexing worker uses this to send requests to the already-running, already-initialized rust-analyzer process.\n\nBuild an LSP indexing worker that:\n1. Watches for `LspDaemonState::Running` on the supervisor\n2. Gets a client handle from the daemon (reuses the existing rust-analyzer process)\n3. Queries `lsp_indexed = 0` files in batches\n4. Sends `textDocument/documentSymbol` for each file\n5. Calls `collect_and_persist_symbols` → sets `lsp_indexed = 1`\n6. On per-file LSP failure, falls back to tree-sitter symbols and still marks `lsp_indexed = 1`\n7. Runs as a background tokio task, independent of the TS worker\n\nFiles:\n- New: `swissarmyhammer-tools/src/mcp/tools/code_context/lsp_worker.rs`\n- Modified: `swissarmyhammer-lsp/src/daemon.rs` (expose client handle to stdin/stdout)\n- Modified: `swissarmyhammer-tools/src/mcp/tools/code_context/mod.rs` (spawn worker)\n\n## Acceptance Criteria\n- [ ] LSP worker reuses the daemon's rust-analyzer process (no second spawn)\n- [ ] LSP worker runs as a background task after MCP client connects\n- [ ] Worker queries `lsp_indexed = 0` files and sends real LSP requests\n- [ ] Worker sets `lsp_indexed = 1` only after receiving LSP response (or fallback)\n- [ ] Worker logs progress and errors\n- [ ] Worker handles daemon restart (re-waits for Running state)\n- [ ] `get status` shows independent progress for `ts_indexed_percent` vs `lsp_indexed_percent`\n\n## Tests\n- [ ] Integration test: worker indexes a known file via rust-analyzer and verifies `lsp_symbols` table has LSP-sourced symbols\n- [ ] Unit test: worker correctly falls back to TS symbols on LSP failure\n- [ ] Unit test: worker waits for daemon Running state before starting"
