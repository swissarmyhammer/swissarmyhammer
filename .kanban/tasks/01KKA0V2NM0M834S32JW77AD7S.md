## What

The spec defines `query ast` — execute tree-sitter S-expression queries against parsed ASTs. This is not implemented. Useful for structural code search (e.g., find all functions with unsafe blocks, all match expressions, etc.).

**Key files:**
- `swissarmyhammer-tools/src/mcp/tools/code_context/mod.rs` — add operation
- `swissarmyhammer-treesitter/` — has tree-sitter parsing, needs query execution

**Approach:**
1. Add `query ast` operation to code_context tool
2. Compile S-expression query with `tree_sitter::Query`
3. Run against parsed files matching language/file filters
4. Return matched captures with file path, line range, captured text
5. Parameters: `query` (S-expression), `language`, `files`, `max_results`
