## What

`lsp_indexed` is never set to 1 in production. The only code that sets it (`mark_lsp_indexed` in `lsp_indexer.rs`) is called from `collect_and_persist_symbols` which is only used in tests. The tree-sitter heuristic path (`ensure_ts_symbols` + `write_ts_edges`) writes synthetic symbols to `lsp_symbols` but never marks files as `lsp_indexed=1`.

**Key files:**
- `swissarmyhammer-tools/src/mcp/tools/code_context/mod.rs` — `index_discovered_files_async` (line 825 sets `ts_indexed` but not `lsp_indexed`)
- `swissarmyhammer-code-context/src/lsp_indexer.rs` — `mark_lsp_indexed` (line 200)

**Approach:** After `ensure_ts_symbols` and `write_ts_edges` succeed for a file, also set `lsp_indexed=1`. The tree-sitter heuristic symbols ARE the LSP symbols until real LSP integration exists.

## Acceptance Criteria
- [ ] Files with tree-sitter-derived symbols get `lsp_indexed=1`
