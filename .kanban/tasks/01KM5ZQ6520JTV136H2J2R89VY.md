## What

Add a leading "selector" column to `DataTable` (`kanban-app/ui/src/components/data-table.tsx`) that renders before all field columns. Each data row gets a selector cell in column 0. The selector cell:

1. Shows a visual indicator — a small dot or checkbox-like element that highlights when the row is the current cursor row
2. Clicking it sets the grid cursor to that row (via existing `onCellClick` / `grid.setCursor`)
3. Right-clicking it triggers the context menu scoped to that row's entity (see card 2)

The selector column is NOT part of the field-based `columns` array — it's a fixed structural column prepended by DataTable itself. The grid cursor's `col` index should still start at 0 for the first field column; the selector is outside the grid's column space (it's a row-level affordance, not a cell).

### Implementation approach

- Add an optional `showRowSelector?: boolean` prop to `DataTableProps` (default `true`)
- Prepend a `<TableHead>` in the header row for the selector column (narrow, ~32px, no label)
- Prepend a `<TableCell>` in each data row with the selector UI
- The selector cell gets the row's `onContextMenu` handler (already exists on the `<TableRow>`)
- For group header rows, add an empty `<TableCell>` or adjust `colSpan` by +1
