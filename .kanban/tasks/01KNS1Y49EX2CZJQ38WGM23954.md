## What
Create `code-context-cli/src/registry.rs` with init/deinit components, mirroring `shelltool-cli/src/registry.rs`.

**Key pattern from shelltool**: `register_all` registers TWO components:
1. `ShelltoolMcpRegistration` (priority 10) — MCP server config
2. `ShellExecuteTool::new()` (priority 20) — tool-specific setup (config file, Bash deny, skill deploy)

For code-context, the pattern is the same:
1. `CodeContextMcpRegistration` (priority 10) — MCP server config
2. `CodeContextTool::new()` (priority 22, already defined) — creates `.code-context/` dir, adds to `.gitignore`

### `CodeContextMcpRegistration` (priority 10)
Registers/unregisters `code-context serve` as MCP server in detected agent configs using mirdan.

```rust
let entry = mirdan::mcp_config::McpServerEntry {
    command: "code-context".to_string(),
    args: vec!["serve".to_string()],
    env: BTreeMap::new(),
};
```

### `CodeContextTool` (priority 22, already implements `Initializable`)
`CodeContextTool` in `swissarmyhammer-tools/src/mcp/tools/code_context/mod.rs` already implements `Initializable`:
- `init()`: creates `.code-context/` directory, adds `.code-context/` to `.gitignore`
- `deinit()`: removes `.code-context/` directory

We just need to register it:
```rust
pub fn register_all(registry: &mut InitRegistry) {
    registry.register(CodeContextMcpRegistration);
    registry.register(CodeContextTool::new());
}
```

Import: `swissarmyhammer_tools::mcp::tools::code_context::CodeContextTool`
