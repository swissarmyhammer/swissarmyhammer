**File:** `swissarmyhammer-leader-election/src/election.rs`\n**Severity:** warning\n\n`LeaderElection`, `LeaderGuard`, and `FollowerGuard` are held inside `Arc<std::sync::Mutex<>>` and sent across `tokio::spawn` boundaries in server.rs. Per the Rust review guidelines, pointer-containing public types must have compile-time `Send`/`Sync` assertions. `File` is `Send` but not `Sync`; `LeaderGuard` contains `_lock_file: File` which is `Send` but not `Sync` — the type is therefore `Send` but not `Sync`. This is likely fine in practice (the guard is only accessed under the mutex), but it's not documented or asserted.\n\n**Fix:** Add:\n```rust\nconst _: () = {\n    fn assert_send<T: Send>() {}\n    fn assert_sync<T: Sync>() {}\n    fn checks() {\n        assert_send::<LeaderElection>();\n        assert_send::<LeaderGuard>();\n        assert_send::<FollowerGuard>();\n    }\n};\n```\nThis catches regressions if a non-Send type is ever added to these structs." #review-finding
