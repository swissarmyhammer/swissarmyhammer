## What
Make `brew install --cask kanban` put the `kanban` CLI on PATH automatically. A Homebrew cask `binary` stanza symlinks a binary from inside the app bundle into `$(brew --prefix)/bin` (already on PATH) at install time — fully automatic, no prompt, no click.

Currently the cask is generated by an inline heredoc in `.github/workflows/release-app.yml` (job `publish-homebrew-cask`), shared by a matrix over `mirdan` and `kanban`. Only `kanban` has a CLI, so the new stanzas must be `kanban`-only.

Files to create/modify:
- `build-support/gen-cask.sh` (new) — extract cask rendering into a script: args `--name`, `--product`, `--version`, `--sha256`, `--dmg-name`, `--desc`, `--homepage`, and an optional `--cli-binary <bundle-relative-path>`. Emits the full cask Ruby to stdout. When `--cli-binary` is given, the output additionally contains:
  - `binary "#{appdir}/<product>.app/Contents/MacOS/kanban"`
  - `conflicts_with formula: "kanban"` (the standalone cargo-dist `kanban` formula is kept everywhere per project decision — `conflicts_with` ensures only one owns `kanban` on PATH at a time).
- `.github/workflows/release-app.yml` — replace the heredoc in `publish-homebrew-cask` with a call to `build-support/gen-cask.sh`. Pass `--cli-binary Contents/MacOS/kanban` for the `kanban` matrix entry only; omit it for `mirdan`. Add a per-app matrix field (e.g. `cli: true`) to drive this.

Reference: the `visual-studio-code` cask uses exactly this pattern (`binary "#{appdir}/Visual Studio Code.app/Contents/Resources/app/bin/code"`).

## Implementation Notes
- `build-support/gen-cask.sh` takes the bare `--version` and reconstructs the release tag as `v<version>` for the download URL, so the script is self-contained (no separate tag argument needed).
- The release workflow now also checks out the `swissarmyhammer` repo at the release tag (into `swissarmyhammer/`) so the script is available — `publish-homebrew-cask` previously only checked out `homebrew-tap`.
- The "Update cask" step builds one `GEN_ARGS` bash array (base args always present, `--cli-binary` conditionally appended). The array is never empty, which keeps `"${GEN_ARGS[@]}"` safe under bash 3.2 (the macOS runner default) where expanding an empty array trips `set -u`.
