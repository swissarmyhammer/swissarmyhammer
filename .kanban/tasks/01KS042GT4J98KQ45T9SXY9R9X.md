## What

Convert every committed plugin bundle in the repo from the `plugin.json` + `entry.ts` layout to TS-only: `index.ts`, no manifest, `name`/`version` as class props.

The six bundles:
- `crates/swissarmyhammer-plugin/examples/plugins/kanban-tasks/`
- `crates/swissarmyhammer-plugin/examples/plugins/file-notes/`
- `crates/swissarmyhammer-plugin/examples/plugins/cli-echo/`
- `crates/swissarmyhammer-plugin/examples/plugins/multi-module/`
- `builtin/plugins/kanban-builtin-probe/`
- `test/builtin/plugins/builtin-probe/`

For each bundle:
- delete `plugin.json`;
- rename `entry.ts` → `index.ts`;
- in the `Plugin` subclass, set `name` and `version` props (from task 2), carrying over the values that were in the deleted manifest;
- `multi-module` keeps its sibling `board-helpers.ts` — only the entry filename changes; update the `./board-helpers.ts` import only if needed (it is a sibling, so the relative specifier is unchanged).
- `cli-echo` keeps the `__CLI_ECHO_COMMAND__` placeholder token — it now lives in `index.ts`.

Also:
- `crates/swissarmyhammer-plugin/examples/plugins/README.md` — rewrite the bundle-layout / manifest section: a bundle is a directory containing `index.ts` (entry by convention); `name`/`version` are `Plugin` class props; the directory name is the plugin's identity; there is no `plugin.json`, no `id`, no `entry` field, no `provides`. Keep the per-example index/sections accurate.
- Fix any stale `plugin.json` / `entry.ts` references in the example-related test files that name the committed bundles: `tests/example_plugins_e2e.rs`, `tests/kanban_tasks_e2e.rs`, `tests/file_notes_e2e.rs`, `tests/cli_echo_e2e.rs`, `tests/multi_module_e2e.rs`, `tests/example_layering_e2e.rs`, and `tests/support/mod.rs` (doc comments / path literals referring to `entry.ts`). Do NOT touch tests that write their own inline `plugin.json` — those are separate tasks.

Discovery (task 1) already loads manifest-less `index.ts` bundles, so once migrated these bundles load through the existing example e2e tests with no test-logic change.
