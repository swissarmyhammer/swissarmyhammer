## What
A batch of `avp-common` tests fail because `ValidatorRunner::load_prompt_library` (crates/avp-common/src/validator/runner.rs) calls `prompt_library.get(".system/validator")` (VALIDATOR_PROMPT_NAME = ".system/validator", defined in crates/avp-common/src/validator/executor.rs), but the `.system/validator` builtin prompt no longer exists.

Root cause: predecessor task 01KTC7HVY5VZZWBW7S7W7FBQFY ("Delete builtin/prompts and stop generating builtin_prompts.rs", status done) deleted `builtin/prompts/` — including `builtin/prompts/.system/validator.md` and `.system/rule.md` — and removed the builtin-prompt embed. `TemplateLibrary::load_all_prompts` (the renamed PromptResolver pipeline) only loads `builtin/_partials/` plus user/local prompt dirs, so the validator system prompt can no longer be resolved.

This is NOT caused by the engine merge/rename task (01KTC7K7GVZ8FXQ4D16ANM0FNJ) — that change was a pure mechanical type/import rename and the same failures occur regardless of the type names. Verified: swissarmyhammer-templating (245 tests + 21 doctests), swissarmyhammer-tools + mirdan (1612), claude-agent + cli lib/bins (1326), llama-agent lib (1059), swissarmyhammer integration all pass green.

## Failing tests (all `.system/validator`-dependent)
- `validator::runner::tests::test_validator_runner_*`
- `validator::executor::tests::test_render_validator_prompt*`, `test_validator_render_context_*`
- integration: `stop_validators_integration`, `recording_replay_integration`, `post_tool_use_integration`, `model_config_integration`, `stop_hook_*`, `validator_tools*_integration`, `validator_block_e2e_integration`

## Fix options
- Move the validator system prompt into a source avp-common owns (e.g. embed `.system/validator` + `.system/rule` as avp builtin validators/partials and resolve via the validator loader), OR re-register them as `_partials`, OR otherwise supply VALIDATOR_PROMPT_NAME without relying on the deleted `builtin/prompts/` tree.
