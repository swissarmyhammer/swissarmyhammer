## What
Add macro support to `swissarmyhammer-operations-macros` so an operation tool declares itself from its operation set and the `io.swissarmyhammer/operations` `_meta` is attached to the tool's `Tool` definition automatically — operation tools become self-describing by construction and `_meta` can never drift from the operation structs.

- In `crates/swissarmyhammer-operations-macros/src/lib.rs`, add the operation-tool analogue of `#[operation]` (e.g. an attribute macro or `operation_tool!` decl macro — pick whichever fits the existing `#[operation]`/`#[param]` style; the `#[operation]` macro is the pattern to mirror).
- The macro takes a tool name, description, and the operation set, and emits code that builds the tool's `Tool` definition with both: the flat `inputSchema` from `generate_mcp_schema` AND `_meta["io.swissarmyhammer/operations"]` from `generate_operations_meta` (the generator added in the operations crate).
- A tool author writes the operation structs + the macro invocation; they never hand-assemble `_meta`.
- Wire format unchanged — `op` stays the single selector; the `tools/call` handler stays an `op` match.

## Acceptance Criteria
- [x] A new macro exists in `swissarmyhammer-operations-macros`, documented with a usage example mirroring the `#[operation]` doc style.
- [x] Code generated by the macro produces a Tool whose `_meta` carries `io.swissarmyhammer/operations` with the noun→verb→parameters tree, and whose `inputSchema` is the existing flat `op`-enum schema.
- [x] `_meta` content is byte-identical to `generate_operations_meta` over the same operation slice (no second source of truth).

## Tests
- [x] A `trybuild`-style or in-crate compile + runtime test: declare a small tool with two operations via the macro, assert the resulting Tool definition has `_meta["io.swissarmyhammer/operations"]["<noun>"]["<verb>"].op` set correctly and `inputSchema.properties.op.enum` lists both ops.
- [x] Test that a tool with no `_meta` operations key is treated as flat (negative control — a flat tool is not mis-tagged).
- [x] Run: `cargo test -p swissarmyhammer-operations-macros` — all green.
