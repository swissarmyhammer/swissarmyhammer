## What

Restructure the review fleet to use the session-fork primitive (task 01KTY8ZV79NX25KJ1BF4CTKPCR / 4ctkpcr) so the shared prompt prefix is decoded ONCE per validator and every batch task forks from it. Today each fan-out task sends one monolithic prompt (change purpose + validator rules + file blocks, `render_fleet_prompt` in crates/swissarmyhammer-validators/src/review/fleet.rs) as a fresh session → on the hybrid Qwen, ~72% of turns pay a full ~15k-token cold prefill (404 trim fallbacks in the 2026-06-12 calcutron run).

New flow in `run_fleet` (fleet.rs) / the pool (crates/swissarmyhammer-validators/src/validators/pool.rs):

- [x] **Prime:** per (review, validator), create a prefix session and prompt it with the shared sections — change purpose + validator mandate/rules/severity + output contract — restructured to end with an explicit "Reply with exactly OK. The files to review arrive in the next message." (a completed turn whose end-of-turn KV lands exactly where continuations begin). Split `render_fleet_prompt` into `render_validator_prefix` and `render_file_payload` accordingly; keep the rendered text byte-stable so all forks share it. *(done: `render_shared_sections` underlies all three renders, so warm and monolithic paths compose from identical bytes)*
- [x] **Confirm + pin:** after the prime turn, verify via the fork-task's observability surface that the KV state is saved (token count > 0) and pin it for the review's duration. If confirmation fails, log and fall back to today's monolithic prompts for that validator (degraded but correct). *(done: `prime_validator_prefix` gates on `saved` + non-zero token count when the backend reports one; backends without token counts — claude CLI — remain forkable per the contract)*
- [x] **Fork per batch task:** each (validator, batch) task = fork the validator's prefix session → prompt with ONLY `render_file_payload(batch)` → parse findings as today. Check the fork response actually attached parent state; degraded fork → log + proceed (it still works, just cold). *(done: `AgentPool::submit_forked` + `collect_forked_task`; a FAILED fork falls back to a monolithic fresh-session prompt — typed `PoolError::ForkFailed` — never a lost task)*
- [x] **Unpin/cleanup** prefix sessions when the validator's batches complete (and on review error paths) so pinned entries don't outlive the review. *(done: `unpin_prefix_session` runs after every batch task resolves, including failures; cancellation-safe since the 2026-06-12 review pass — the pin is held by a `SessionPinGuard` whose `Drop` releases it even when the fan-out future is dropped mid-collect)*
