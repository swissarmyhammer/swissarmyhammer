## What\nRefactor `FilesTool` from a unit struct to an instance with configurable operations. This lets us create a read-only variant for validators without a whole new tool.\n\n**Files:**\n- `swissarmyhammer-tools/src/mcp/tools/files/mod.rs` — main refactor\n- `swissarmyhammer-tools/src/mcp/tools/files/description.md` — may become unnecessary\n\n**Approach:**\n1. Add instance fields to `FilesTool`:\n   - `operations: Vec<&'static dyn Operation>` — subset of FILE_OPERATIONS\n   - `agent_only: bool` — controls `is_agent_tool()` return value\n2. Add constructors:\n   - `FilesTool::all()` — all 5 ops, `agent_only: true` (current behavior)\n   - `FilesTool::read_only()` — only read+glob, `agent_only: false`\n3. Change `description()` to aggregate from operation descriptions + examples instead of the static `description.md` file\n4. Change `schema()` to use `self.operations` instead of `&FILE_OPERATIONS`\n5. Change `execute()` to check `self.operations` before dispatching — reject ops not in the set\n6. Update `register_file_tools()` to use `FilesTool::all()`\n\n## Acceptance Criteria\n- [ ] `FilesTool::all()` produces identical behavior to current (backward compat)\n- [ ] `FilesTool::read_only()` only allows read+glob, rejects write/edit/grep\n- [ ] `description()` aggregates from individual operation descriptions\n- [ ] `schema()` only advertises operations in the instance's set\n- [ ] `is_agent_tool()` returns the configured `agent_only` value\n\n## Tests\n- [ ] Unit test: all() schema includes all 5 ops\n- [ ] Unit test: read_only() schema includes only read+glob\n- [ ] Unit test: read_only() execute rejects write/edit\n- [ ] Unit test: description() includes operation descriptions\n- [ ] `cargo test -p swissarmyhammer-tools`"
