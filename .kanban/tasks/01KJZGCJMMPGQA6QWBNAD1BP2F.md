## Mental Model\n\n**Focus IS a CommandScope.** Clicking or navigating determines which CommandScope is focused. The moniker is just the address — the scope is the thing.\n\nEverything flows from the focused scope:\n- Palette shows commands from focused scope's chain\n- Keybindings resolve from focused scope's chain\n- Context menu collects from focused scope's chain\n- Visual elements ask \"is the focused scope me or a descendant of me?\"\n\n## Registry\n\n`EntityFocusProvider` gains a `Map<string, CommandScope>` ref (not state — no re-renders on register/deregister). Each FocusScope registers `(moniker, scope)` on mount, deregisters on unmount.\n\nThe CommandScope already has a `.parent` pointer built by React's nesting. Registering the leaf gives access to the full chain. Add `moniker?: string` to CommandScope so ancestor walks can identify which scopes are FocusScope-created.\n\n## Hooks\n\n- `useFocusedScope(): CommandScope | null` — looks up focused moniker in registry, returns that scope (with full parent chain) or null\n- `useIsFocused(moniker): boolean` — true if focused scope IS this moniker or has this moniker in its ancestor chain\n\n## Wiring\n\nKeyBindingHandler and CommandPalette use `useFocusedScope()` to resolve commands, falling back to root AppShell scope when null.\n\n## Unfocus\n\nClick on empty board background → `setFocus(null)` → focused scope is null → fall back to app scope.\n\n## Subtasks\n\n- [ ] Add `moniker?: string` to CommandScope interface\n- [ ] FocusScope passes moniker when creating its CommandScope\n- [ ] Add `scopeRegistry: Map<string, CommandScope>` ref to EntityFocusProvider\n- [ ] Add `registerScope` / `unregisterScope` to context value\n- [ ] FocusScope registers on mount, deregisters on unmount (useEffect)\n- [ ] Add `useFocusedScope()` hook\n- [ ] Add `useIsFocused(moniker)` hook with ancestor chain walk\n- [ ] Wire KeybindingHandler to resolve from focused scope (fallback to root)\n- [ ] Wire CommandPalette to show commands from focused scope\n- [ ] Add unfocus: click on board background calls setFocus(null)\n- [ ] Update data-focused to use useIsFocused (columns highlight when child focused)\n- [ ] Tests: registry lifecycle, useFocusedScope, useIsFocused ancestor detection, unfocus\n- [ ] Run tests to verify no regressions
