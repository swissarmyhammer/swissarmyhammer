## What

Selecting a board from the NavBar dropdown doesn't switch the board. The `BoardSelector` component (`board-selector.tsx:93-97`) dispatches `file.switchBoard` directly via `useDispatchCommand` instead of calling its `onSelect` prop:\n\n```tsx\nonValueChange={(path) => {\n  dispatchSwitchBoard({ args: { path } }).catch(console.error);\n}}\n```\n\nThis bypasses `handleSwitchBoard` in `App.tsx:528-542`, which is the function that:\n1. Sets `activeBoardPath` React state (drives the entire UI)\n2. Sets `activeBoardPathRef.current` (used by board-changed listener)\n3. Dispatches `file.switchBoard` with explicit `windowLabel: WINDOW_LABEL`\n4. Calls `refresh()` to reload board data and entities\n\nWithout calling `onSelect`, the frontend never updates `activeBoardPath` or reloads entities. The backend UIState updates, `board-changed` fires, and the listener at `App.tsx:444` tries to detect the change — but this is a fragile async roundtrip that depends on path format matching between UIState and `activeBoardPathRef`.\n\n**Additionally**, the `dispatchSwitchBoard` call doesn't pass `windowLabel` in args, so `SwitchBoardCmd` (`file_commands.rs:40-45`) defaults to `\"main\"` — broken for secondary windows.\n\n### Fix\n\nIn `board-selector.tsx`, call `onSelect(path)` in the `onValueChange` handler instead of dispatching directly:\n\n```tsx\nonValueChange={(path) => {\n  onSelect(path);\n}}\n```\n\n`handleSwitchBoard` already handles the full switch: React state update, backend dispatch with correct `windowLabel`, and data refresh. Remove `dispatchSwitchBoard` and its `useDispatchCommand` import if no longer used.\n\n**Files to modify:**\n- `kanban-app/ui/src/components/board-selector.tsx` — `onValueChange` handler (line 95-97): call `onSelect(path)` instead of `dispatchSwitchBoard`; remove unused `useDispatchCommand(\"file.switchBoard\")` import\n\n## Acceptance Criteria\n- [ ] Selecting a different board from the dropdown switches to that board immediately\n- [ ] Board data (tasks, columns, entities) refreshes after switching\n- [ ] Secondary windows switch correctly (not always targeting \"main\")\n- [ ] The `onSelect` prop is the single path for board switching\n\n## Tests\n- [ ] Update or add test in `kanban-app/ui/src/components/board-selector.test.tsx`: selecting a board from the dropdown calls `onSelect` with the correct path\n- [ ] `cd kanban-app/ui && pnpm vitest run board-selector` — all pass\n- [ ] `cd kanban-app/ui && pnpm vitest run` — no regressions
