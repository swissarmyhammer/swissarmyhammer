## What
Implement the `delete` cells of the domain grid (`ideas/expect.md` §"Operations"): `expectation delete`, `observation delete`, `golden delete`.

- In `crates/swissarmyhammer-expect/src/ledger.rs` + `tools/expect/mod.rs`:
  - `expectation delete <scope>` — remove the `*.expect.md` spec AND its received observation AND its golden (the full identity-mirrored set). Per the design example flow ("remove spec + its observation + golden").
  - `observation delete <scope>` — remove only the `.expect/received/<path>.received.json`.
  - `golden delete <scope>` — remove only the `.expect/goldens/<path>.golden.json`.
  - Resolve scope via the loader; operate on the mirrored paths. Safe: report what was removed; missing files are a no-op note, not an error.

## Acceptance Criteria
- [ ] `expectation delete src/checkout/coupon` removes the spec, its received, and its golden.
- [ ] `observation delete` / `golden delete` remove only their respective artifact.
- [ ] Deleting a non-existent artifact reports cleanly (no panic, no hard error).
- [ ] Each op returns a structured summary of what was removed.

## Tests
- [ ] `crates/swissarmyhammer-expect/src/ledger.rs` tests on a fixture repo: full delete removes all three; scoped deletes remove only one; missing-artifact no-op.
- [ ] Tools op tests for the three delete ops.
- [ ] `cargo nextest run -p swissarmyhammer-expect delete` passes.
