## What

When a board column has enough cards to scroll, the scrollbar overlaps the right edge of the cards. The scroll container at `kanban-app/ui/src/components/column-view.tsx:239` uses `overflow-y-auto px-2` — the horizontal padding is inside the scroll container, so the scrollbar renders on top of the card content rather than outside it.

**Fix**: Add `scrollbar-gutter: stable` to the scroll container so the scrollbar space is always reserved, preventing the overlap. This is a one-line CSS change.

Alternatively, move `pr-2` to be larger (e.g. `pr-4`) only when scrollbar is present, but `scrollbar-gutter: stable` is the cleaner approach and works across all platforms.

### File to modify
- `kanban-app/ui/src/components/column-view.tsx` — line ~239, add `scrollbar-gutter-stable` (Tailwind arbitrary value: `[scrollbar-gutter:stable]`) to the scroll container className
