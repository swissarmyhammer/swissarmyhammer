## What

`active_view_id` is duplicated in AppConfig.windows and UIState. The `set_active_view` Tauri command bypasses dispatch_command. The YAML commands `nav.view.*` exist but the frontend also calls the direct Tauri command.

### Changes
- Wire `nav.view` / `nav.view.*` command implementations to mutate UIState
- Remove `set_active_view` Tauri command
- Frontend: replace `invoke(\"set_active_view\")` with `dispatch_command`
- Frontend: read from `useUIState().activeViewId` instead of `get_ui_context`
- Remove `active_view_id` from AppConfig.windows / WindowState (UIState persists it)

## Acceptance Criteria
- [ ] No direct `invoke(\"set_active_view\")` in frontend
- [ ] View changes go through `dispatch_command`
- [ ] Active view persists across app restarts
- [ ] `set_active_view` Tauri command removed
