Build the frontend view system: provider, left nav, command-based view switching, and data-driven command attachment.\n\n## Key files\n- `ui/src/types/kanban.ts` — add ViewDef, ViewCommandDef TypeScript types\n- `ui/src/lib/views-context.tsx` — ViewsProvider, useViews() hook\n- `ui/src/components/left-nav.tsx` — icon nav generated from view registry\n- `ui/src/components/app-shell.tsx` — add nav.view + per-view alias commands\n- `ui/src/App.tsx` — wrap with ViewsProvider, add left nav, conditional view render\n- `ui/src/components/board-view.tsx` — refactor to read commands from ViewDef\n\n## Command Attachment Pattern\nThis is the key design: ViewDef declares commands as data (id, name, keys), renderers attach execute functions.\n\n```typescript\n// In BoardView (or any view renderer):\nconst executeMap: Record<string, () => void> = {\n  \"board.newCard\": () => { /* add card logic */ },\n  \"board.collapseAll\": () => { /* collapse logic */ },\n  \"board.expandAll\": () => { /* expand logic */ },\n};\n\n// Merge ViewDef command metadata with local execute fns\nconst commands: CommandDef[] = viewDef.commands.map(cmd => ({\n  id: cmd.id,\n  name: cmd.name,\n  keys: cmd.keys,\n  execute: executeMap[cmd.id],\n  available: cmd.id in executeMap, // unavailable if no implementation\n}));\n\n// Register via CommandScopeProvider\n<CommandScopeProvider commands={commands}>\n  {/* view content */}\n</CommandScopeProvider>\n```\n\nFlow:\n1. ViewsProvider loads ViewDefs from backend (includes command metadata)\n2. Active view renders (e.g. BoardView)\n3. BoardView reads commands from its ViewDef\n4. Creates CommandDef[] by merging YAML metadata + local execute fns\n5. Wraps children in CommandScopeProvider\n6. Palette sees view commands via scope chain\n7. On view switch: old scope exits, new scope enters\n\n## ViewsProvider\n- Fetches views via `invoke('list_views')` on mount\n- Manages `activeViewId` state (defaults to first view)\n- Exposes: `views`, `activeView`, `setActiveView`, `loading`\n\n## Left nav\n- Maps view registry to vertical icon strip (lucide icons by name)\n- Click dispatches `nav.view` command with view name\n- Active view highlighted\n- Always visible, collapsed to icons\n\n## Checklist\n- [ ] Add ViewDef, ViewCommandDef types to kanban.ts\n- [ ] Create ViewsProvider with useViews() hook\n- [ ] Create LeftNav component\n- [ ] Add nav.view command + per-view aliases to AppShell\n- [ ] Refactor BoardView to read commands from ViewDef and attach implementations\n- [ ] Integrate into App.tsx layout (ViewsProvider + LeftNav + conditional render)\n- [ ] App launches with left nav showing board icon\n- [ ] :view board in palette works\n- [ ] View commands appear in palette with correct names/keys from YAML\n\n#phase2-views
