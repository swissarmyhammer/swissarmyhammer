## What
The authoring op a coding agent drives to capture intent at the moment it's expressed. Per `ideas/expect.md` §"expect expectation create" and §"Errors that teach".

- New `crates/swissarmyhammer-expect/src/create.rs` (+ tools op):
  - `create(source, intent_text) -> drafted spec(s)`: read the intent-bearing artifact, draft one or more `*.expect.md` files with explicit intent + bounded (~3-5) criteria, loop each through `doctor` until every field is green (the agent patches exactly the red fields using the structured per-field diagnostics — it can't emit an invalid spec), record a candidate observation, and leave the result **unapproved** (ledger state `new`) for a human to confirm.
  - Hand the agent the RESOLVED frontmatter schema (closed enums + live `model:` set) and the authoring rules: intent-is-mandatory, keep-criteria-bounded, state-the-right-reason (the 401-vs-200 defense), prefer invariants over literals.
  - Sources (all feed one draft→doctor→confirm pipeline): `--from-chat` (default interactive), `--from-task <id>` (kanban task acceptance criteria, link back as provenance only), `--from-spec <path>` (mine should/must/example), `--from-session` (capture a hand-verified run), and a bare intent string.
- Wire `expectation create` op in `tools/expect/mod.rs`; it uses the agent seam (driving agent) to author. Leaves file + unapproved candidate observation.
- (Authoring SKILL that proactively offers capture mid-conversation is a separate skill task; this is the tool it calls.)

## Acceptance Criteria
- [ ] `expect expectation create "<intent>"` drafts a `*.expect.md`, loops it through doctor until all fields are Ok, and leaves it unapproved (`new`).
- [ ] The agent is handed the resolved schema incl. the live model set; a draft can never be left in a doctor-Error state.
- [ ] `--from-task <id>` drafts from a kanban task's criteria and records task provenance (tag/comment) without coupling the spec to the task lifecycle.
- [ ] `--from-spec`/`--from-session`/`--from-chat` route through the same pipeline.

## Tests
- [ ] Integration test with a stub authoring agent: from an intent string, produces a doctor-green spec left in `new` state with a candidate observation.
- [ ] Test the doctor-repair loop: a deliberately-bad first draft is corrected to green.
- [ ] `cargo nextest run -p swissarmyhammer-expect create` passes.
