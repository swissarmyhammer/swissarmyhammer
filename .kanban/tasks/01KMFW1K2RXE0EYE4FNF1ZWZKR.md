title: Add progress-ring display type for percent_complete field

## What\n\nThe `percent_complete` field on board uses `display: number`, but `NumberDisplay` has a hack (lines 9-15 in `number-display.tsx`) that sniffs for `{ total, completed, percent }` objects and delegates to `ProgressDisplay` (a linear bar). Meanwhile the navbar uses a bespoke `BoardProgress` component (`board-progress.tsx`) with a Recharts radial ring that's completely outside the field system.\n\nThis conflates two display concerns: a plain number and a radial progress visualization. A general number field should not render a ring.\n\n**Fix**: Create a `progress-ring` display type that renders the radial chart, and register it in the field system.\n\n1. Create `kanban-app/ui/src/components/fields/displays/progress-ring-display.tsx` — extract the radial chart rendering from `board-progress.tsx` into a proper display component that accepts `DisplayProps`\n2. Create `kanban-app/ui/src/components/fields/registrations/progress-ring.tsx` — register `progress-ring` display\n3. Update `kanban-app/ui/src/components/fields/registrations/index.ts` — import the new registration\n4. Update `swissarmyhammer-kanban/builtin/fields/definitions/percent_complete.yaml` — change `display: number` to `display: progress-ring`\n5. Remove the `{ total, completed, percent }` sniffing hack from `number-display.tsx` (lines 9-15)\n\n## Acceptance Criteria\n- [ ] `percent_complete` field renders as a radial ring via the field system\n- [ ] `NumberDisplay` only renders plain numbers — no object sniffing\n- [ ] `display: progress-ring` is a registered display type\n- [ ] Both compact and full modes render appropriately\n\n## Tests\n- [ ] `kanban-app/ui/src/components/fields/displays/progress-ring-display.test.tsx` — renders ring for `{ done: 3, total: 5, percent: 60 }`, returns null for total 0\n- [ ] Run: `cd kanban-app/ui && npm test` — no regressions"
