## What
The single entry point both consumers (the `diagnostics` tool and the `files edit` op) call. In `swissarmyhammer-diagnostics`:

- `async fn diagnose(session, paths, config) -> DiagnosticsReport`: ensure each path's document is open/synced via the shared `LspSession`, run the settle engine, map to `DiagnosticRecord`s, apply severity filter + per-report cap.
- **Report sharply** (design "compute freely, surface selectively"): always include the edited/queried file; of its dependents, include only those that *actually broke*. Compute the one-hop blast radius via code-context (`get_blastradius` / inbound callers) — blast radius is cheap, the saved turn is not — then run diagnostics on those dependents and keep only files with new error/warning-severity diagnostics, ranked and capped. No project-wide dump, no unrelated standing warnings.
- This introduces a `swissarmyhammer-diagnostics → swissarmyhammer-code-context` dependency for blast radius (allowed: diagnostics sits above code-context).
- Respect `dependents?` toggle and `severity?`/`settle_ms?` overrides in config.

## Depends on
- "Settle/debounce engine for diagnostics quiescence"
- "Rewire code-context onto the shared LSP session; collapse its get_diagnostics" (shared session + code-context consuming it)

## Acceptance Criteria
- [ ] `diagnose(paths)` returns a `DiagnosticsReport` for the given files, settled, severity-filtered, capped.
- [ ] Edited file always present; only *broken* one-hop dependents folded in, ranked + capped; clean dependents excluded.
- [ ] `dependents?`/`severity?`/`settle_ms?` honored.

## Tests
- [ ] `cargo test -p swissarmyhammer-diagnostics`: with a mock session emitting known diagnostics + a stub blast-radius, assert the report includes the target and only broken dependents, respects the cap, and drops clean dependents. Model-free, <1s.
- [ ] Integration (gated on rust-analyzer): a fixture crate where editing file A breaks dependent B; assert `diagnose([A])` reports A's error and B-as-broken, and does NOT report an unrelated clean file C.
