`sah init` should stop denying Bash. The deny moves to serve (paired card); init goes back to agent-agnostic. Cleanup stays owned by deinit.

## Change
- Remove `DenyBash` from the canonical `Initializable` set in `apps/swissarmyhammer-cli/src/commands/registry.rs` so `sah init` no longer calls `mirdan::install::deny_tool` across all detected agents.
- Retain the **cleanup** path: `sah deinit` must still re-allow Bash (`mirdan::install::allow_tool`) so a previously serve-applied deny can be torn down. Decide the shape: keep `DenyBash` as a deinit-only component (no `init()` effect), or fold the allow into deinit teardown another way. (See `components/mod.rs:118-158` for the current symmetric component.)
- Keep the lifecycle the user chose: serve writes idempotently; init/deinit owns removal.
