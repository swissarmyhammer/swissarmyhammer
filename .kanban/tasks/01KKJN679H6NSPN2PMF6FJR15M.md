**File:** `swissarmyhammer-tools/src/mcp/server.rs` lines ~412–446\n**Severity:** warning\n\nThe re-election loop breaks as soon as promotion succeeds or when `ws_lock.is_leader()` is true. Once exited it never re-runs. This means:\n\n1. If the promoted leader's `LeaderGuard` is later dropped for any reason (e.g. another call to `try_promote` on the same workspace inadvertently drops the guard), the process silently becomes leaderless with no recovery.\n2. If `start_indexing_workers_after_promotion` panics and the spawned task aborts, the workspace still holds the leader lock but no workers are running — there is no watchdog.\n\nThe loop is also polling every 5 seconds unconditionally. A `tokio::sync::Notify` or `watch` channel from `LeaderGuard::dropped` would be cleaner.\n\n**Fix:** At minimum, document that the loop is one-shot (only promotes once). For production resilience, keep the loop alive to handle unexpected demotion." #review-finding
