**File:** `swissarmyhammer-code-context/src/workspace.rs` lines 131–135\n**Severity:** warning\n\nWhen `elect()` returns `Follower`, the code immediately attempts to open the database read-only:\n\n```rust\nlet db = Connection::open_with_flags(&db_path, SQLITE_OPEN_READ_ONLY | ...)?;\n```\n\nBut if this is the very first run, the leader hasn't had a chance to create `index.db` yet (it does so on the same path, moments later in its own branch). The follower's `open_with_flags` call will fail with \"unable to open database file\" because SQLite read-only open does not create the file.\n\nThe test `test_second_open_is_reader` masks this because both opens happen in the same process where the leader branch runs first. In a real multi-process scenario with tight timing the follower will get an `Err`.\n\n**Fix:** Retry the follower DB open with a short backoff (or wait for the DB file to appear), or have the leader create the DB file before returning from `elect()` via a pre-creation step." #review-finding
