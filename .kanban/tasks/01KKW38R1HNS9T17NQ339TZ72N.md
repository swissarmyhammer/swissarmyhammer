title: refresh() closes over stale activeBoardPathRef — missing from useCallback deps
---
kanban-app/ui/src/App.tsx:127-151\n\n`refresh` is defined with `useCallback(async () => { ... }, [])` — an empty dependency array. It reads `activeBoardPathRef.current` inside, which is intentional (the ref pattern avoids stale closures). However, the function also calls `setActiveBoardPath` and `setEntitiesByType`, which are stable setState setters and fine.\n\nThe subtle issue: `refresh` calls `refreshBoards(activeBoardPathRef.current)` and then, on success, updates `activeBoardPathRef.current = active.path` directly. This direct mutation of the ref inside the callback happens concurrently with React's render cycle. If `refresh` is called twice in quick succession (e.g. `board-changed` fires and then an `entity-created` fires a column refresh immediately after), the second call may read a partially-updated ref value.\n\nThis is a minor race rather than a data-loss bug, but the comment on line 128 ("activeBoardPathRef.current") should note why the empty deps array is intentional, to prevent future maintainers from adding the ref to the array (which would break the pattern).\n\nSuggestion: Add a comment `// intentional: reads ref to avoid stale closure, ref is kept in sync below` adjacent to the useCallback declaration."
