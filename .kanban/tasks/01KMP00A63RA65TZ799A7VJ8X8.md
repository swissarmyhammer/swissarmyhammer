**File:** `kanban-app/ui/src/lib/entity-focus-context.tsx:58-83`\n\n**What:** `invokeFocusChange` is declared as a plain `function` inside the `EntityFocusProvider` component body. It is then called from `setFocus`, `pushClaim`, `updateClaim`, and `popClaim` -- all of which are wrapped in `useCallback(..., [])` with empty dependency arrays. Because `invokeFocusChange` is recreated on every render but the callbacks that reference it are memoized with `[]` deps, the callbacks always close over the *first* render's version of `invokeFocusChange`.\n\n**Why this matters:** Today this is safe by accident -- `invokeFocusChange` only reads from `registryRef` (a ref, stable across renders) and calls `invoke` (a module-level import). It has no dependency on component state. But this is fragile: if anyone adds state or prop references to `invokeFocusChange` in the future, stale closures will silently bite. The lint suppression is implicit (no eslint comment, the deps just happen to be empty).\n\n**Suggestion:** Either (a) move `invokeFocusChange` outside the component body (it only needs the registry ref, which can be a parameter), or (b) wrap it in `useCallback` with explicit deps and list it as a dep of the other callbacks, or (c) store it in a ref. Option (a) is simplest and communicates intent best." #review-finding
