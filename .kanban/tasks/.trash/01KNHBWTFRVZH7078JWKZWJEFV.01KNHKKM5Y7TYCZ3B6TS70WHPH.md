## What\n\nThe \"Add Task\" button on columns appears to do nothing because the `entity-created` event handler in `rust-engine-container.tsx` (line 205-254) always re-fetches the entity via `get_entity` IPC call, ignoring the `fields` payload that the backend now provides (from the drag-and-drop fix in `flush_and_emit_for_handle`).\n\nThe re-fetch path adds latency and may fail in edge cases (e.g. if the entity file isn't fully flushed when `get_entity` is called). The `entity-field-changed` handler already has a fast path that uses `fields` from the event directly — the `entity-created` handler should do the same.\n\n### Root cause\n\nIn `kanban-app/ui/src/components/rust-engine-container.tsx`, the `entity-created` listener (line 205) destructures only `{ entity_type, id, board_path }` from the event payload, ignoring the `fields` property that is now enriched by the backend. It then always calls `invoke(\"get_entity\", ...)` for a round-trip re-fetch.\n\n### Fix\n\nWhen `event.payload.fields` is non-empty, construct the `Entity` directly from the event payload (same as the `entity-field-changed` fast path) and add it to the store synchronously. Fall back to `get_entity` re-fetch only when `fields` is empty or missing.\n\n### Files to modify\n\n1. **`kanban-app/ui/src/components/rust-engine-container.tsx`** (entity-created listener, line ~205)\n   - Destructure `fields` from `event.payload`\n   - When `fields` is non-empty (`Object.keys(fields).length > 0`): construct entity as `{ id, entity_type, fields }` and call `setEntitiesFor` directly — no IPC round-trip\n   - When `fields` is empty: fall back to existing `get_entity` re-fetch path\n   - Keep the `column` → full refresh special case as-is\n\n## Acceptance Criteria\n\n- [ ] Clicking \"Add Task\" on a column immediately shows a new task card in that column\n- [ ] The new task has correct default title and position\n- [ ] Creating tasks in rapid succession doesn't lose any (no race conditions)\n- [ ] Column creation still triggers full refresh (existing special case preserved)\n- [ ] `cd kanban-app/ui && npx vitest run` passes\n\n## Tests\n\n- [ ] `kanban-app/ui/src/components/rust-engine-container.test.tsx` — add test: entity-created event with populated fields adds entity to store without IPC call\n- [ ] `kanban-app/ui/src/components/rust-engine-container.test.tsx` — add test: entity-created event with empty fields falls back to get_entity re-fetch\n- [ ] Run `cd kanban-app/ui && npx vitest run` — all pass\n\n## Workflow\n- Use `/tdd` — write failing tests first, then implement to make them pass.
