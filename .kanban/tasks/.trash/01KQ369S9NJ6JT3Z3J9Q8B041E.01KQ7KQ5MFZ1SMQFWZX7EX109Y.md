## What

**REWRITE** `llama-agent/` against the new ACP 0.11.1 SDK design. The old `impl Agent` pattern is gone (see spike findings on task 01KQ367HE0Z8ZSXY90CTT8QYGG and migration guide https://agentclientprotocol.github.io/rust-sdk/migration_v0.11.x.html).

504 lines reference `agent_client_protocol`. Largest single hot-spot is `acp/server.rs` (the `AcpServer` `impl Agent`) and `acp/translation.rs` (heavy `match` over schema enums).

## Spike-confirmed scope

### The full Agent impl is gone
- `llama-agent/src/acp/server.rs` defines `AcpServer` with the old 9-method `impl Agent`. **Rewrite as a builder graph.**
- Supporting modules (`acp/session.rs`, `acp/translation.rs`, `acp/mcp_client_factory.rs`, `acp/permissions.rs`, `acp/commands.rs`, `acp/terminal.rs`, `acp/filesystem.rs`, `acp/plan.rs`, `acp/error.rs`, `acp/config.rs`, `acp/mod.rs`) all need import-path migration to `agent_client_protocol::schema::*` plus call-site adjustments where they construct/consume the old types.
- `llama-agent/src/mcp_client_handler.rs`, `llama-agent/src/mcp.rs` — same.
- `llama-agent/src/agent.rs` (top-level agent server) — same.
- `llama-agent/src/examples/acp_stdio.rs` — rewrite the stdio entrypoint to use the new `Agent.builder().connect_to(...)` pattern; the existing stdio harness is an `impl Agent` driver.

### Match-arm rewrites
- `acp/translation.rs` has heavy `match` over `SessionUpdate`, `ContentBlock`, `ContentChunk`, `ToolKind`, `ToolCallStatus`, `StopReason`. Each block needs:
  - Import-path migration to `schema::`
  - Continued explicit handling of `#[non_exhaustive]` variants
  - Possibly a structural rewrite if the new typed message dispatch (`MatchDispatchFrom`, `if_request_from`, `if_message_from`) is a better fit than open-coded match.

### Schema type usages (504 unique refs)
All the following move from `agent_client_protocol::X` to `agent_client_protocol::schema::X`:
`AgentCapabilities`, `AuthenticateRequest`, `AuthenticateResponse`, `AvailableCommand`, `AvailableCommandInput`, `CancelNotification`, `ClientCapabilities`, `ContentBlock`, `ContentChunk`, `CurrentModeUpdate`, `Error`/`ErrorCode`/`Result`, `ExtNotification`, `ExtRequest`, `ExtResponse`, `FileSystemCapabilities`, `Implementation`, `InitializeRequest`/`InitializeResponse`, `LoadSessionRequest`/`LoadSessionResponse`, `McpCapabilities`, `McpServer{Http,Sse,Stdio}`, `NewSessionRequest`/`NewSessionResponse`, `Plan`, `PlanEntry`, `PlanEntryPriority`, `PlanEntryStatus`, `PromptCapabilities`, `PromptRequest`/`PromptResponse`, `ProtocolVersion::{V0,V1}`, `RawValue::from_string`, `ReadTextFileRequest`, `SessionId`, `SessionMode`, `SessionModeId`, `SessionModeState`, `SessionNotification`, `SessionUpdate::{AgentMessageChunk, CurrentModeUpdate, Plan, ToolCall, ToolCallUpdate, UserMessageChunk}`, `SetSessionModeRequest`/`SetSessionModeResponse`, `StopReason::{Cancelled, EndTurn, MaxTokens, Refusal}`, `TextContent`, `ToolCall`, `ToolCallId`, `ToolCallStatus::Pending`, `ToolCallUpdate`, `ToolKind`, `WriteTextFileRequest`. (Note: `Error`, `ErrorCode`, `Result` and the role markers are still re-exported at the crate root — those don't need the path change.)

### Tests
- `llama-agent/tests/acp_integration.rs` and `llama-agent/tests/integration/acp_*.rs` drive `AcpServer` end-to-end. **Rewrite the test harness** alongside the production rewrite — these tests are the regression net for the AcpServer rebuild.
