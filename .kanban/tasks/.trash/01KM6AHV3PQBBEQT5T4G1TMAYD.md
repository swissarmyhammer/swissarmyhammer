## What\n\nExpose archive/unarchive operations through the MCP kanban tool so that `archive task`, `unarchive task`, and `list archived` work as MCP operations alongside `delete task`, `complete task`, etc.\n\n### 1. Add verbs/nouns to types\n\nIn `swissarmyhammer-kanban/src/types/operation.rs`:\n- Add `Archive` and `Unarchive` to the `Verb` enum\n- Add `Archived` to the `Noun` enum (for `list archived`)\n- Add `as_str()` arms, `from_alias()` arms, and `Display` implementations\n- Aliases: `archive` for Archive, `unarchive`/`restore` for Unarchive\n\n### 2. Add ArchiveTask operation with `#[operation]` macro\n\nThe existing `ArchiveTask` in `swissarmyhammer-kanban/src/task/archive.rs` was written as a plain struct without the `#[operation]` macro. It needs to be updated to use the macro so it gets verb/noun metadata for MCP schema generation:\n\n```rust\n#[operation(verb = \"archive\", noun = \"task\", description = \"Archive a task\")]\n```\n\nSimilarly, add `UnarchiveTask` and `ListArchived` operations.\n\n### 3. Wire dispatch\n\nIn `swissarmyhammer-kanban/src/dispatch.rs`, add match arms:\n- `(Verb::Archive, Noun::Task)` → `ArchiveTask`\n- `(Verb::Unarchive, Noun::Task)` → `UnarchiveTask` (new operation that calls `ectx.unarchive()`)\n- `(Verb::List, Noun::Archived)` → `ListArchived` (new operation that calls `ectx.list_archived()`)\n\n### 4. Register in schema\n\nIn `swissarmyhammer-kanban/src/schema.rs`, add `ArchiveTask`, `UnarchiveTask`, and `ListArchived` to `KANBAN_OPERATIONS` so they appear in MCP tool descriptions.\n\n### 5. Update parse layer\n\nIn `swissarmyhammer-kanban/src/parse/mod.rs` (or wherever op strings are parsed), ensure `\"archive task\"`, `\"unarchive task\"`, and `\"list archived\"` parse correctly.\n\n### Files\n- `swissarmyhammer-kanban/src/types/operation.rs` — add Archive, Unarchive verbs + Archived noun\n- `swissarmyhammer-kanban/src/task/archive.rs` — add `#[operation]` macro, add UnarchiveTask\n- `swissarmyhammer-kanban/src/task/mod.rs` — export new types\n- `swissarmyhammer-kanban/src/dispatch.rs` — add dispatch arms\n- `swissarmyhammer-kanban/src/schema.rs` — register operations\n- `swissarmyhammer-kanban/src/parse/mod.rs` — if needed for parsing\n\n## Acceptance Criteria\n- [ ] `{\"op\": \"archive task\", \"id\": \"...\"}` archives a task via MCP\n- [ ] `{\"op\": \"unarchive task\", \"id\": \"...\"}` unarchives a task via MCP\n- [ ] `{\"op\": \"list archived\"}` returns all archived tasks via MCP\n- [ ] Archived tasks don't appear in `list tasks` or `next task`\n- [ ] Schema generation includes the new operations\n\n## Tests\n- [ ] `dispatch_archive_task` — parse + dispatch integration test\n- [ ] `dispatch_unarchive_task` — parse + dispatch integration test\n- [ ] `dispatch_list_archived` — parse + dispatch integration test\n- [ ] `cargo test -p swissarmyhammer-kanban`
