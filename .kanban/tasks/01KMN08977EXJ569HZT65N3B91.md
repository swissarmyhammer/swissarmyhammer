## What

`file.closeBoard` silently fails every time. The board stays in the open boards list and cannot be removed.

### Root cause

Two bugs in the close board flow:

**Bug 1 ŌĆö No `path` arg passed to Rust:**
The frontend command definition (`kanban-app/ui/src/components/app-shell.tsx:292`) has no `execute` handler and no `args`. `dispatchCommand` sends `{ cmd: "file.closeBoard", boardPath: ... }` to Rust. But `CloseBoardCmd::execute` (`swissarmyhammer-kanban/src/commands/file_commands.rs:84-89`) reads `ctx.args.get("path")` ŌĆö which is empty because `boardPath` is NOT injected into `ctx.args`. It always returns `MissingArg("path")`.

**Bug 2 ŌĆö Path canonicalization mismatch:**
Even if the path were passed, `CloseBoardCmd::execute` calls `ui.remove_open_board(&path)` with the raw string, while the dispatch handler at `kanban-app/src/commands.rs:943-950` also calls `state.close_board(&target)` which canonicalizes. If the stored path is canonical but the arg isn't (or vice versa), one or both removals fail silently.

**Bug 3 ŌĆö Boards with no path (trashed entries):**
A board entry in the config with no path or an empty path has no way to be closed. The close command requires a path to identify the board. Need to handle boards with missing/empty paths ŌĆö remove them from the open list without trying to close a handle.

### Fix

**Option A (simplest):** Give the frontend command an `execute` handler that reads `boardPath` from context and passes it as `args: { path: boardPath }`:

```tsx
// app-shell.tsx, file.closeBoard command
execute: async () => {
  const path = boardPathRef.current;
  if (path) {
    await invoke("dispatch_command", {
      cmd: "file.closeBoard",
      args: { path },
      windowLabel: getCurrentWindow().label,
    });
  }
},
```

**Option B (more robust):** Have `CloseBoardCmd` fall back to `boardPath` from the command context when `args.path` is not provided, and canonicalize consistently.

**For trashed entries:** Add cleanup logic in `auto_open_board` or board loading that removes entries with empty/missing paths from the persisted config instead of trying to open them.

### Files to modify

- **Modify**: `kanban-app/ui/src/components/app-shell.tsx` ŌĆö add `execute` handler to `file.closeBoard` command that passes `boardPath` as `args.path`
- **Modify**: `swissarmyhammer-kanban/src/commands/file_commands.rs` ŌĆö canonicalize path in `CloseBoardCmd::execute` before calling `remove_open_board`
- **Modify**: `kanban-app/src/state.rs` ŌĆö in `auto_open_board`, skip or remove entries with empty/missing paths
