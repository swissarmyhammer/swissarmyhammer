swissarmyhammer-kanban/src/schema.rs:23-83\n\nThe new `KANBAN_OPERATIONS` static in schema.rs is a near-duplicate of the `KANBAN_OPERATIONS` in `swissarmyhammer-tools/src/mcp/tools/kanban/mod.rs:96-148`. Both use `Box::leak` / `Lazy` to create static operation instances. The MCP tool version uses `once_cell::sync::Lazy` with pre-allocated statics; the kanban crate version uses `LazyLock` with `Box::leak`. If an operation is added/removed, both lists must be updated.\n\nSuggestion: Have the MCP tool call `swissarmyhammer_kanban::schema::kanban_operations()` instead of maintaining its own list."
