## What
Scaffold a NEW leaf crate `crates/swissarmyhammer-search/` — a generic, in-memory document ranker with NO workspace-sibling dependencies (true leaf). It is consumed by BOTH `swissarmyhammer-code-context` (chunk search) and `swissarmyhammer-kanban` (task search), so it must NOT depend on `model-embedding`, `swissarmyhammer-embedding`, or any sibling crate.

DO NOT hand-roll the word/identifier splitter. Use existing, dependency-light crates already resolved in the workspace `Cargo.lock`:
- `convert_case` (0.10) — `convert_case::split(identifier, boundaries) -> Vec<&str>` with the `Boundary` enum. Handles camelCase/PascalCase, acronyms (`HTTPRequest` -> `http`,`request`), and digit boundaries. We choose the `Boundary` set, so digit-splitting is a config choice, not code.
- `unicode-segmentation` (1.13) — `UnicodeSegmentation::unicode_words()` (UAX #29) for the punctuation/whitespace/Unicode word layer (zero transitive deps).

Files to create:
- `crates/swissarmyhammer-search/Cargo.toml` — package `swissarmyhammer-search`, `version.workspace = true`, edition/license/repository `.workspace = true`. Deps: `serde = { workspace = true }` (for `Serialize` on `Hit`/`Signals`), `convert_case = { workspace = true }`, `unicode-segmentation = { workspace = true }`. No tokio, no model crates, no sibling crates.
- `crates/swissarmyhammer-search/src/lib.rs` — module decls + the public types below.
- `crates/swissarmyhammer-search/src/tokenize.rs` — the tokenizer (wrapping the two crates) + `char_trigrams`.

Workspace wiring (edit `Cargo.toml` at repo root):
- Add `"crates/swissarmyhammer-search"` to `[workspace] members`.
- Add `swissarmyhammer-search = { path = "crates/swissarmyhammer-search" }` to `[workspace.dependencies]` (alphabetical, near the other `swissarmyhammer-*` entries).
- Ensure `convert_case` and `unicode-segmentation` are present in `[workspace.dependencies]` (add them pinned to the already-resolved versions `convert_case = "0.10"`, `unicode-segmentation = "1"` if not already listed — they exist transitively in `Cargo.lock` today).

Public types (in `lib.rs`; this card defines them, the scoring card implements `search()`):
- `pub struct Doc { pub id: String, pub fields: Vec<Field>, pub embedding: Option<Vec<f32>> }`
- `pub struct Field { pub weight: f32, pub text: String }`
- `pub struct Query { pub text: String, pub embedding: Option<Vec<f32>>, pub weights: SignalWeights, pub top_k: usize, pub min_score: Option<f32> }`
- `pub struct SignalWeights { pub w_bm25: f32, pub w_trigram: f32, pub w_cosine: f32 }` with a `Default` of all `1.0`.
- `pub struct Hit { pub id: String, pub score: f32, pub signals: Signals }` — derive `Debug, Clone, serde::Serialize`.
- `pub struct Signals { pub bm25: f32, pub trigram: f32, pub cosine: f32 }` — derive `Debug, Clone, Copy, serde::Serialize`.

Tokenizer (`tokenize.rs`, pure functions, no DB/embedding access — NO Porter stemming; trigram carries fuzziness):
- `pub fn tokenize(text: &str) -> Vec<String>`:
  - Pass `text` through `unicode_words()` to get word runs (strips punctuation/whitespace, Unicode-correct).
  - For each word, apply `convert_case::split(word, &BOUNDARIES)` to break camelCase/PascalCase/acronym/underscore boundaries.
  - Define a module-level `BOUNDARIES` slice (the agreed decision): include case + acronym + delimiter boundaries (`LowerUpper`, `UpperLower`/`Acronym`, `Underscore`, `Hyphen`, `Space`); EXCLUDE the digit boundaries so `sha256`/`utf8`/`base64` stay whole. Document the exact slice in the docstring.
  - Lowercase each segment; preserve duplicates (BM25 needs term frequency — do NOT dedup); no empty strings.
- `pub fn char_trigrams(s: &str) -> Vec<[char; 3]>` over a lowercased string — sliding length-3 windows of `chars()` (Unicode-safe). Strings shorter than 3 chars return an empty Vec. (This is n-gram windowing, not a splitter — keep inline.)

This is foundational and reusable: BM25 (corpus df/avgdl + per-doc tf) and trigram-Dice both consume `tokenize`/`char_trigrams` in the scoring card.
