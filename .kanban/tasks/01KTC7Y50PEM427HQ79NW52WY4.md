## What
Add a code-aware tokenizer that the BM25 and trigram signals of the fused `search code` will share. New module: `crates/swissarmyhammer-code-context/src/ops/search_fusion/tokenize.rs` (create the `search_fusion` submodule dir; register `pub mod search_fusion;` inside `crates/swissarmyhammer-code-context/src/ops/search_code.rs` OR add `pub mod search_fusion;` to `crates/swissarmyhammer-code-context/src/ops/mod.rs` — pick the latter so the primitives are reusable and unit-testable independently of the op).

Tokenization rules (agreed design — do NOT add Porter stemming; trigram carries fuzziness):
- Lowercase everything.
- Split on whitespace and punctuation (anything not `[A-Za-z0-9]`).
- Additionally split identifiers on camelCase boundaries (`getUser` -> `get`, `user`) and snake_case boundaries (`get_user` -> `get`, `user`).
- Digit runs split off as their own tokens at letter/digit transitions is optional — keep simple: treat `[A-Za-z0-9]+` runs then sub-split on camelCase. Document the exact rule chosen in the function docstring.

Public API (pure, no DB):
- `pub fn tokenize(text: &str) -> Vec<String>` — full pipeline above.
- `pub fn char_trigrams(s: &str) -> Vec<[char; 3]>` (or `Vec<String>` of length-3 windows) over a lowercased string — used by the trigram-Dice signal. Document handling of strings shorter than 3 chars (return empty).

This is foundational: BM25 (corpus df/avgdl + per-chunk tf) and trigram-Dice both consume `tokenize`. No DB access, no embedding access — pure string functions.

## Acceptance Criteria
- [ ] `tokenize("getUserById")` yields `["get", "user", "by", "id"]` (camelCase split, lowercased).
- [ ] `tokenize("get_user_by_id")` yields `["get", "user", "by", "id"]` (snake_case split).
- [ ] `tokenize("fn parse_config() -> Result")` splits punctuation and keywords into lowercase word tokens with no empty strings.
- [ ] `char_trigrams("get_user")` returns the sliding length-3 windows of the lowercased string; strings shorter than 3 chars return an empty Vec.
- [ ] Module is reachable from `search_code.rs` (it `use`s `super::search_fusion::tokenize`).
