## What

Delete the `FallbackFocusZoneBody` (kanban-app/ui/src/components/focus-zone.tsx, lines ~530–648) and `FallbackFocusScopeBody` (kanban-app/ui/src/components/focus-scope.tsx, lines ~449–558) branches and the conditional rendering that selects them. These exist solely so a `<FocusZone>` or `<FocusScope>` mounted outside the spatial provider stack (no `<SpatialFocusProvider>`, no `<FocusLayer>`) silently degrades to a plain `<div>` instead of throwing.

The focus system exists to work. Mounting a primitive without its providers is a setup bug, not a supported mode. The fallback hides those bugs by treating them as no-error and is rationalized only by tests that skip the provider stack — production code never enters this branch (per the existing comments).
