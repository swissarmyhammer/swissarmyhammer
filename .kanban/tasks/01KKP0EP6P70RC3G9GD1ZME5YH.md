## What\nMove the CLI generation and argument extraction utilities out of `swissarmyhammer-cli` (where they're private methods on `DynamicCli`/`CliExecutor`) into `swissarmyhammer-tools` so that `kanban-cli` can reuse them without duplicating code.\n\n**Functions to extract (from `swissarmyhammer-cli/src/dynamic_cli.rs`):**\n- `CommandData`, `ArgData` structs (lines ~1050-1070)\n- `precompute_tool_command(tool: &dyn McpTool)` → `Option<CommandData>`\n- `create_command_data_from_tool()`, `create_noun_command_data()`, `create_verb_command_data()`\n- `build_tool_subcommand_from_data(name, data)` → `clap::Command`\n- `precompute_args(schema)` → `Vec<ArgData>`\n- `SchemaParser::parse_arg_data()` and related schema→arg conversion\n\n**Functions to extract (from `swissarmyhammer-cli/src/cli_executor.rs`):**\n- `extract_noun_verb_arguments(matches, schema)` → `Map<String, Value>`\n- `build_arguments_from_matches(matches, op_string, schema)` → `Map<String, Value>`\n- `extract_value_from_matches(matches, name, schema)` → `Option<Value>`\n\n**Target location:** `swissarmyhammer-tools/src/cli_gen.rs` (new module)\n\n**Modify:** `swissarmyhammer-cli/src/dynamic_cli.rs` and `cli_executor.rs` — call the shared functions instead of private methods\n\n## Acceptance Criteria\n- [ ] `CommandData`, `ArgData`, CLI generation, and arg extraction are public in `swissarmyhammer-tools`\n- [ ] `sah tool kanban` still works identically (same CLI structure, same execution)\n- [ ] Functions are usable from outside `swissarmyhammer-cli`\n\n## Tests\n- [ ] Existing `sah` CLI tests pass: `cargo nextest run -p swissarmyhammer-cli`\n- [ ] `cargo nextest run -p swissarmyhammer-tools` passes
