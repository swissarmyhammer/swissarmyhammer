## What

Add the runtime dependencies needed to support `serve`, `init`/`deinit`, and `doctor` to `kanban-cli/Cargo.toml`.

**Runtime dependencies** (add to `[dependencies]`):
- `rmcp` — MCP stdio server runtime
- `swissarmyhammer-common` — `InitRegistry`, `InitScope`, `CliReporter`, `lifecycle`
- `swissarmyhammer-skills` — `SkillResolver` for loading the builtin kanban skill on init
- `swissarmyhammer-templating` — render `{{version}}` in the deployed skill SKILL.md
- `mirdan` — agent detection, MCP config registration, `deploy_skill_to_agents`/`uninstall_skill`
- `swissarmyhammer-doctor` — `DoctorRunner`, `Check`, `CheckStatus`

No `[build-dependencies]` needed — there is no separate `cli.rs` or `build.rs` for this project. The lifecycle commands are added directly to the existing schema-driven command tree in `main.rs`, not through a parallel clap `Parser`.

Note: do NOT add `swissarmyhammer-directory` — kanban uses `".kanban"` as a plain string literal.

Reference: `shelltool-cli/Cargo.toml` shows the exact workspace keys to use.
