## What

Add the dependencies needed to support `serve`, `init`/`deinit`, `doctor`, and build-time doc generation to `kanban-cli/Cargo.toml`.

**Runtime dependencies** (add to `[dependencies]`):
- `rmcp` — MCP stdio server runtime
- `swissarmyhammer-common` — `InitRegistry`, `InitScope`, `CliReporter`, `lifecycle`, `health`
- `mirdan` — agent detection + MCP config registration
- `swissarmyhammer-doctor` — `DoctorRunner`, `Check`, `CheckStatus`

**Build dependencies** (add `[build-dependencies]` section, same as shelltool-cli):
- `clap` (workspace)
- `clap-markdown` (workspace)
- `clap_mangen` (workspace)
- `clap_complete` (workspace)

Note: do NOT add `swissarmyhammer-directory` — kanban uses `".kanban"` as a plain string literal; `ShellConfig`/`DirectoryConfig` are shell-tool abstractions that don't apply here.

Reference: `shelltool-cli/Cargo.toml` shows the exact workspace keys to use.

## Acceptance Criteria
- [ ] `cargo build -p kanban-cli` succeeds with new dependencies declared
- [ ] `[build-dependencies]` section present with the four clap doc-gen crates
