## What

Update the `entity-created` handler in `rust-engine-container.tsx` to use the `fields` from the event payload when available (the watcher already populates these for new files). Only fall back to `get_entity` when `fields` is empty (e.g. the entity was created by a store event before the watcher detected it).\n\nThis is NOT the same as the enrichment anti-pattern. The watcher reads the new file from disk to build its cache and naturally has the fields. The backend isn't doing an extra read — the fields are a byproduct of cache population.\n\n### Current behavior\n\n`entity-created` handler (line 205-254) always calls `invoke(\"get_entity\", ...)` regardless of whether the event payload has fields.\n\n### Fix\n\n1. Destructure `fields` from `event.payload`\n2. If `fields` has entries (`Object.keys(fields).length > 0`): construct entity directly as `{ id, entity_type, fields }` and add to store — no IPC round-trip\n3. If `fields` is empty: fall back to `get_entity` (entity created by store before watcher cache populated)\n4. Keep the `column` → full refresh special case as-is\n\n### Files to modify\n\n1. **`kanban-app/ui/src/components/rust-engine-container.tsx`** (entity-created listener, ~line 205)\n   - Destructure `fields` from event payload\n   - Add fast path when fields is populated\n   - Keep `get_entity` as fallback for empty fields\n\n## Acceptance Criteria\n\n- [ ] `entity-created` with populated fields adds entity without IPC round-trip\n- [ ] `entity-created` with empty fields falls back to `get_entity`\n- [ ] \"Add Task\" button immediately shows new task card\n- [ ] Column creation still triggers full refresh\n- [ ] `cd kanban-app/ui && npx vitest run` passes\n\n## Tests\n\n- [ ] `kanban-app/ui/src/components/rust-engine-container.test.tsx` — test: entity-created with populated fields adds entity without invoke call\n- [ ] `kanban-app/ui/src/components/rust-engine-container.test.tsx` — test: entity-created with empty fields calls get_entity\n- [ ] Run `cd kanban-app/ui && npx vitest run` — all pass\n\n## Workflow\n- Use `/tdd` — write failing tests first, then implement to make them pass. #events
