**Problem**: ValidatorExecutorLink::process (validator_executor.rs:269) serializes the entire PostToolUseInput as JSON via `serde_json::to_value(input)`. For Edit tools this includes `tool_result.originalFile` (entire file), duplicated old/new strings in both `tool_input` and `tool_result`. Diffs computed by PostToolUseFileTracker sit in ChainContext unused.\n\n**Solution**: Build a pre-formatted markdown string instead of raw JSON.\n\n### Subtasks\n\n1. **New function** `format_validator_context(input: &serde_json::Value, diffs: Option<&[FileDiff]>) -> String`\n   - For Edit/Write tools (diffs available): compact YAML header (tool, file, session, cwd) + fenced ```diff block. Strip originalFile, oldString, newString, structuredPatch, and raw tool_input old/new strings entirely.\n   - For other tools: full context as fenced ```yaml block (not JSON).\n   - Place in `avp-common/src/turn/diff.rs` (already has `format_diffs_fenced`).\n\n2. **Update `ValidatorExecutorLink::process`** (validator_executor.rs)\n   - After `serde_json::to_value(input)`, read `ctx.get::<Vec<FileDiff>>(CTX_FILE_DIFFS)`\n   - Call formatting function to produce pre-formatted string\n   - Pass string as hook_context value (wrap in serde_json::Value::String)\n\n3. **Update prompt templates** (both `.system/validator.md` and `.system/rule.md`)\n   - Change ```` ```json\\n{{ hook_context }}\\n``` ```` to just `{{ hook_context }}` — no wrapping fences, Rust produces the right fences per content type.\n\n4. **Tests**: unit tests for the formatting function covering Edit, Write, Bash, and MCP tool inputs.\n\n**Key files**: avp-common/src/turn/diff.rs, avp-common/src/chain/links/validator_executor.rs, builtin/prompts/.system/validator.md, builtin/prompts/.system/rule.md
