avp-common/src/turn/state.rs:206-244\n\nThe `clear()` method takes a `session_id` parameter and correctly removes only that session's on-disk state file. However, line 209 calls `cache.clear()` on the entire `content_cache` Mutex, which wipes stashed pre-content for ALL sessions, not just the target session.\n\nIn a concurrent scenario where a parent session and a subagent session share the same TurnStateManager (via Arc), clearing the subagent's state would destroy the parent's in-flight pre-content cache. This contradicts the session-isolation design stated in the module docstring.\n\nSuggestion: The content_cache is keyed by tool_use_id, not session_id, so there is no clean way to scope the clear. Either:\n1. Re-key the cache as `HashMap<(session_id, tool_use_id), ...>` and clear only the matching session entries, or\n2. Document that clear() is only safe to call when no other sessions are actively using PreToolUse/PostToolUse (which may be true in practice since clear is called at SessionStart when the old session is done)." #review-finding
