## What
Add the middle rung of the verdict ladder: tolerance bands that decide a criterion the deterministic tier couldn't, before any model call. Per `ideas/expect.md` §"The Verdict Ladder" Tier 2 and §"How outcomes get validated" (2).

- In `crates/swissarmyhammer-expect/src/evaluate.rs`:
  - Tier 2 resolution for criteria not decided by Tier 1: embedding cosine similarity vs the golden anchor (pinned embedder, default threshold 0.80 from config / `similarity_threshold` override), numeric tolerance, and Levenshtein for near-string matches.
  - Reuse the project's embedder seam (the `EmbedderFactory`/`TextEmbedder` used by review — `tools/review/review_op.rs` `default_embedder_factory`; pin the embedding model per config `[embedder]`). The embedder checkpoint is recorded in the golden (reproducibility).
  - Compare-by-tier: tolerance ⇒ golden's score + band; drift if the score leaves the band.
- Gate: Tier 2 only runs on the residual Tier 1 couldn't decide; it short-circuits before Tier 3.

## Acceptance Criteria
- [ ] A criterion that exact-match would fail but is semantically equivalent (cosine ≥ threshold) passes at Tier 2.
- [ ] A numeric value within tolerance passes; outside drifts.
- [ ] Tier 2 runs only after Tier 1 is undecided (assert ordering); the pinned embedder/threshold come from config and are recorded in the golden.
- [ ] A score that leaves the golden's band is reported as drift, not a hard fail.

## Tests
- [ ] `crates/swissarmyhammer-expect/src/evaluate.rs` Tier-2 tests with a stub/deterministic embedder: equivalent-reword passes, numeric tolerance, band-exit drift.
- [ ] `cargo nextest run -p swissarmyhammer-expect tier2` passes.
