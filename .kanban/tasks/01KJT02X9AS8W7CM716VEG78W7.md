**File:** `swissarmyhammer-kanban-app/ui/src/components/fields/field-placeholder.tsx`, lines 30-36\n\n**What:** When a user presses Enter to commit an edit, the input loses focus, which triggers the `onBlur` handler. Both `onKeyDown(Enter)` and `onBlur` call `onCommit()`, resulting in a double commit of the same field value. Similarly, pressing Escape fires `onCancel()` but then `onBlur` fires `onCommit()`, overriding the cancellation.\n\n**Why:** This causes `onUpdateField` to be called twice per edit (once on Enter, once on blur), which means the IPC `update_entity_field` command fires twice. Worse, on Escape the user intends to cancel but the blur handler commits the value anyway.\n\n**Suggestion:** Track commit/cancel state in a ref so onBlur can short-circuit if the edit was already resolved by a keyboard event. A common pattern:\n```tsx\nconst committedRef = useRef(false);\n// onKeyDown Enter: committedRef.current = true; onCommit(...);\n// onKeyDown Escape: committedRef.current = true; onCancel();\n// onBlur: if (!committedRef.current) onCommit(...);\n```\n\n- [ ] Add a ref to track whether commit/cancel already happened\n- [ ] Guard onBlur to skip if ref is already set\n- [ ] Add a test case: press Escape then verify onUpdateField is NOT called (the existing Escape test passes because fireEvent doesn't trigger blur, but a real browser will)\n- [ ] Add a test case: press Enter then verify onUpdateField is called exactly once" #warning
