`mirdan/src/new.rs:243-248`\n\nThe `run_new_tool` global path is:\n```rust\nxdg_base_dir(\"XDG_DATA_HOME\", \".local/share\")\n    .join(\"tools\")\n    .join(name)\n```\n\nThis puts tools under `~/.local/share/tools/`, with no product namespace prefix. By contrast `run_new_validator` uses `AvpConfig::XDG_NAME` to produce `~/.local/share/avp/validators/`, and `cli.rs` documents `$XDG_DATA_HOME/tools/` in help text.\n\nThe lack of a product namespace means tools from this application will collide in `~/.local/share/tools/` with any other tool that places files there (an unscoped shared directory). For validators the name is `avp/validators/` which is scoped. Tools should similarly be `sah/tools/` or similar.\n\nSuggestion: Change to `SwissarmyhammerConfig::XDG_NAME` or a product-specific constant, e.g.:\n```rust\nxdg_base_dir(\"XDG_DATA_HOME\", \".local/share\")\n    .join(SwissarmyhammerConfig::XDG_NAME)\n    .join(\"tools\")\n    .join(name)\n``` #review-finding
