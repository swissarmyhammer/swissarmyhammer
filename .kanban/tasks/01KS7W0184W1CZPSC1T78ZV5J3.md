#elicitation

## Context / Why
MCP/ACP elicitation requests carry a JSON-Schema-like `ElicitationSchema` describing form fields. The panel must render these "as documented in MCP" and coerce the user's input back into a typed `CreateElicitationResponse`. Keeping the schema→form logic as a pure, React-free module makes every field/type/validation rule exhaustively unit-testable (same split as `conversation.ts`'s pure reducer).

Read the property-schema types in `apps/kanban-app/ui/node_modules/@agentclientprotocol/sdk/dist/schema/types.gen.d.ts`:
- `ElicitationPropertySchema` variants by `type`: `string` (`StringPropertySchema` — may carry `enum`/`oneOf` for single-select, `format`, `minLength`/`maxLength`), `number`, `integer`, `boolean`, `array` (`MultiSelectPropertySchema` — multi-select).
- `ElicitationSchema`: `{ properties?: {[k]: ElicitationPropertySchema}, required?: string[], title?, description? }`.
- `ElicitationContentValue` = `string | number | boolean | string[]`.

## What
Create `apps/kanban-app/ui/src/ai/elicitation.ts`:
- [x] `type ElicitationField` descriptor: `{ key, label, kind: "text" | "textarea" | "select" | "boolean" | "number" | "integer" | "multiselect", required, options?, description?, format? }` and a discriminator for url-mode vs form-mode requests.
- [x] `parseElicitation(request: CreateElicitationRequest)`: returns either `{ mode: "form", message, fields: ElicitationField[] }` or `{ mode: "url", message, url, elicitationId }`. Map each property schema to a field kind (string+enum/oneOf → select; string (long/format) → textarea vs text; array → multiselect; boolean → boolean; number/integer → number/integer).
- [x] `initialFormState(fields)`: default values per field.
- [x] `validateForm(fields, values)`: returns missing-required / type errors.
- [x] `toAcceptResponse(fields, values): CreateElicitationResponse` with `action:"accept"` and a `content` map of `ElicitationContentValue` (coerce numbers, booleans, string[]).
- [x] `declineResponse()` / `cancelResponse()` helpers returning the respective `CreateElicitationResponse`.

## Acceptance Criteria

## Implementation notes
- Created `apps/kanban-app/ui/src/ai/elicitation.ts` (pure, React-free) and `apps/kanban-app/ui/src/ai/elicitation.test.ts` (38 tests, all green in the browser vitest project).

- `ElicitationField` carries an optional `default` (in the field's editing type) so `initialFormState(fields)` seeds from the schema without re-reading the request.
