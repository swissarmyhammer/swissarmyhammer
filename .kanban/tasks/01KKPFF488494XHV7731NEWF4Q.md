## What
The frontend computes `blockedIds` in `board-view.tsx` by iterating all tasks and checking dependencies — the exact same logic the backend already runs in `enrich_all_task_entities()` which sets `ready: bool` on every task.

Remove the client-side `blockedIds` useMemo and instead read `entity.fields.ready === false` to determine if a card should be dimmed.

**Files:**
- `kanban-app/ui/src/components/board-view.tsx` — remove `blockedIds` useMemo, stop passing it as prop
- `kanban-app/ui/src/components/column-view.tsx` — remove `blockedIds` prop, use `!getStr(entity, "ready")` or check `entity.fields.ready`
- `kanban-app/ui/src/components/entity-card.tsx` — `isBlocked` stays as a prop but derived from entity data
- `kanban-app/ui/src/components/sortable-task-card.tsx` — same

## Acceptance Criteria
