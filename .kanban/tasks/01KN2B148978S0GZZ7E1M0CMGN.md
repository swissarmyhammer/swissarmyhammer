## What

`update_window_title` in `kanban-app/src/commands.rs:49` hardcodes a string format using `handle.ctx.name()` — a filesystem path stem. Meanwhile, `{{entity.display_name}}` already exists in `scope_commands.rs` via `TemplateParams` and `resolve_name_template`. The window title should use the board entity's display name — the same data everything else uses.

There is no fallback needed. The board entity's `name` field is set to the directory name during `init board`. If there's a board, it has a display name. If there's no board, the title is just the app name.

### Fix

Replace `update_window_title`'s `board_name: Option<&str>` (currently fed by `handle.ctx.name()`, a path stem) with reading the board entity's display name from the entity context. The board entity's `search_display_field: name` in `board.yaml` declares which field is the display name.

- `kanban-app/src/commands.rs` — refactor `update_window_title` to read the board entity's display name from the entity context
- Callers at `create_window_impl` (~line 756), `dispatch_command_internal` BoardSwitch (~line 1070), BoardClose (~line 1095), and `restore_windows` (~line 789) — pass the entity context instead of `handle.ctx.name()`
