## What\n\nMake `dispatch_command` persist the inspector stack to config whenever a `ui.inspect`, `ui.inspector.close`, or `ui.inspector.close_all` command executes. Also restore the persisted stack into `UIState` on app startup.\n\nThis replaces the separate `set_inspector_stack` Tauri command — persistence becomes a side-effect of the command itself.\n\n### Files\n- `kanban-app/src/commands.rs` — after executing UI commands, detect `InspectorStack` in result and persist to config. Remove `set_inspector_stack` function.\n- `kanban-app/src/main.rs` — remove `set_inspector_stack` from handler registration. On startup, load persisted stack into `UIState`.\n- `swissarmyhammer-commands/src/ui_state.rs` — add `set_inspector_stack` method for initialization.\n\n### Approach\n1. In `dispatch_command`: after `cmd_impl.execute()`, if the result deserializes as `UIStateChange::InspectorStack(stack)`, save `stack` to `config.windows[label].inspector_stack` and `config.save()`.\n2. Add `UIState::set_inspector_stack(&self, stack: Vec<String>)` method.\n3. During app startup (after config loads), call `ui_state.set_inspector_stack(config.windows[\"main\"].inspector_stack)`.\n4. Delete `set_inspector_stack` Tauri command and its handler registration.\n\n## Acceptance Criteria\n- [ ] Running `ui.inspect` via dispatch_command persists the stack to config\n- [ ] Running `ui.inspector.close` persists the updated stack\n- [ ] `UIState` is populated from config on startup\n- [ ] `set_inspector_stack` Tauri command removed\n- [ ] `cargo nextest run` passes\n\n## Tests\n- [ ] `swissarmyhammer-commands/src/ui_state.rs` — unit test for `set_inspector_stack`\n- [ ] `cargo nextest run -p swissarmyhammer-commands` passes\n- [ ] `cargo nextest run -p kanban-app` passes"
